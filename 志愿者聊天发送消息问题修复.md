# 志愿者聊天发送消息问题修复

## 问题描述

用户报告：志愿者在聊天界面里，连内容都发不出来。

## 问题分析

### 症状
- 志愿者可以输入文字
- 发送按钮可以点击（不是灰色）
- 但是点击发送按钮后，消息没有发送出去
- 没有任何错误提示

### 根本原因

在 `ChatPage.ets` 的 `handleSend()` 方法中，有一个检查条件：

```typescript
handleSend() {
  const text: string = this.inputText.trim();
  if (!text || !this.targetUserId || this.isSending) {
    return;  // 如果 targetUserId 为空，直接返回，不发送消息
  }
  // ... 发送消息的代码
}
```

问题在于：
1. 发送按钮的 `.enabled()` 条件只检查 `!this.isSending && this.inputText.trim().length > 0`
2. 但是 `handleSend()` 方法还额外检查 `!this.targetUserId`
3. 如果 `targetUserId` 为空，按钮虽然可以点击，但消息不会发送

### 可能导致 targetUserId 为空的原因

1. **路由参数缺失**: 从其他页面跳转到聊天页面时，没有传递必要的参数
2. **API 返回数据不完整**: `RequestApi.getRequestDetail()` 返回的数据中 `userId` 字段为空
3. **角色判断错误**: `resolveChatTarget()` 方法中的逻辑判断有问题

## 解决方案

### 1. 添加调试日志

在关键方法中添加 `console.info` 和 `console.warn` 日志，帮助诊断问题：

#### resolveChatTarget() 方法
```typescript
private resolveChatTarget(request: HelpRequest): void {
  console.info(`resolveChatTarget - currentUserRole: ${this.currentUserRole}, currentUserId: ${this.currentUserId}`);
  console.info(`resolveChatTarget - request.userId: ${request.userId}, request.volunteerId: ${request.volunteerId}`);
  
  if (this.currentUserRole === 'help-seeker' || this.currentUserId === request.userId) {
    // 当前用户是求助者，目标用户是志愿者
    if (!this.targetUserId) {
      this.targetUserId = request.volunteerId ? request.volunteerId : '';
    }
    if (!this.targetUserName || this.targetUserName === '\u804a\u5929') {
      this.targetUserName = request.volunteerName ? request.volunteerName : '\u5fd7\u613f\u8005';
    }
    this.pageTitle = this.targetUserName;
    console.info(`resolveChatTarget - Set target as volunteer: ${this.targetUserId}`);
    return;
  }

  // 当前用户是志愿者，目标用户是求助者
  this.targetUserId = request.userId ? request.userId : '';
  this.targetUserName = request.userName ? request.userName : '\u6c42\u52a9\u8005';
  this.pageTitle = this.targetUserName;
  console.info(`resolveChatTarget - Set target as help-seeker: ${this.targetUserId}`);
}
```

#### handleSend() 方法
```typescript
handleSend() {
  const text: string = this.inputText.trim();
  console.info(`handleSend - text: "${text}", targetUserId: "${this.targetUserId}", isSending: ${this.isSending}`);
  
  if (!text || !this.targetUserId || this.isSending) {
    console.warn(`handleSend - Cannot send: text=${!!text}, targetUserId=${!!this.targetUserId}, isSending=${this.isSending}`);
    return;
  }

  this.isSending = true;
  console.info(`handleSend - Sending message to ${this.targetUserId} for request ${this.requestId}`);

  MessageApi.sendMessage({
    requestId: this.requestId,
    receiverId: this.targetUserId,
    text: text,
    type: 'text'
  }).then((message: ChatMessage) => {
    // ... 成功处理
  }).catch((err: Error) => {
    console.error('send message failed:', err.message || err);
  }).finally(() => {
    this.isSending = false;
  });
}
```

### 2. 修复空值处理

在 `resolveChatTarget()` 方法中，添加空值保护：

```typescript
// 修改前
this.targetUserId = request.userId;

// 修改后
this.targetUserId = request.userId ? request.userId : '';
```

这样即使 `request.userId` 为 `null` 或 `undefined`，也会设置为空字符串，避免后续判断出错。

## 常见问题排查

### 问题 1: targetUserId 始终为空

**可能原因**:
- 后端 API 返回的数据中 `userId` 字段为 `null`
- 数据库中该字段没有值

**解决方法**:
1. 检查后端日志，确认 API 返回的数据
2. 检查数据库 `help_requests` 表，确认 `user_id` 字段有值
3. 运行 SQL 查询：
   ```sql
   SELECT id, user_id, volunteer_id, title FROM help_requests WHERE id = 'req001';
   ```

### 问题 2: 角色判断错误

**可能原因**:
- `currentUserRole` 的值不是预期的 'volunteer'
- 用户信息存储有问题

**解决方法**:
1. 检查登录后存储的用户信息：
   ```typescript
   const user = await StorageUtil.get<UserProfile>('user');
   console.info('Current user:', JSON.stringify(user));
   ```
2. 确认 `user.role` 的值是 'volunteer'

### 问题 3: 路由参数传递错误

**可能原因**:
- 从志愿者主页跳转到聊天页面时，没有传递正确的参数

**解决方法**:
检查跳转代码，确保传递了 `requestId`:
```typescript
router.pushUrl({
  url: 'pages/volunteer/ChatPage',
  params: {
    requestId: request.id,  // 必须传递
    // volunteerId 和 volunteerName 是可选的
  }
});
```
