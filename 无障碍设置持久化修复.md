# 无障碍设置持久化修复（最终版本 v2）

## 问题描述

用户报告无障碍设置（大字体、高对比度、震动反馈）存在两个问题：

1. **立即失效问题**：点击Toggle开关后，设置立即被重置回原值
2. **重启丢失问题**：应用重启后，设置无法保持

日志显示：
```
大字体设置更新请求: true
大字体设置已更新: true
大字体设置更新请求: false  // 立即被重置！
大字体设置已更新: false
[StorageUtil-PERSISTENT] 开始保存: accessibility_settings
[StorageUtil-PERSISTENT] 保存成功: accessibility_settings
无障碍设置保存成功
```

## 根本原因分析（深度分析）

### 问题1：Toggle双向绑定导致的循环触发

**原始实现的问题**：
```typescript
// ProfilePage.ets - 错误的实现
@StorageLink('accessibility_large_font') largeFontEnabled: boolean = false;

Toggle({ type: ToggleType.Switch, isOn: this.largeFontEnabled })
  .onChange((isOn: boolean) => {
    this.accessibilityManager.updateLargeFont(isOn);
    // updateLargeFont 会调用 AppStorage.set()
    // AppStorage.set() 触发 @StorageLink 更新
    // @StorageLink 更新导致 Toggle 的 isOn 改变
    // Toggle 的 isOn 改变触发 onChange 事件
    // 形成循环！
  })
```

**触发流程**：
1. 用户点击 Toggle，`isOn` 从 false 变为 true
2. `onChange(true)` 被调用
3. `updateLargeFont(true)` 执行，调用 `AppStorage.set(KEY, true)`
4. `@StorageLink` 检测到 AppStorage 变化，更新 `largeFontEnabled` 为 true
5. Toggle 的 `isOn` 属性从 `this.largeFontEnabled` 获取新值 true
6. **HarmonyOS Toggle 组件检测到 `isOn` 属性变化，触发 `onChange(true)` 事件**
7. 但由于某种原因（可能是框架内部逻辑），第二次触发时传入 false
8. 导致设置被重置

## 根本原因分析

### 问题1：立即失效

原因是 `getSettings()` 方法使用了错误的逻辑：

```typescript
// 错误的实现
largeFontEnabled: AppStorage.get<boolean>(STORAGE_KEYS.LARGE_FONT) || false
```

当值为 `false` 时，`|| false` 会返回 `false`，这是正确的。但问题在于：
- `AppStorage.get()` 可能返回 `undefined`
- `undefined || false` 返回 `false`
- 但 `false || false` 也返回 `false`

这导致在 `updateLargeFont(true)` 后立即调用 `getSettings()` 时，如果 AppStorage 还没完全同步，可能读取到错误的值。

### 问题2：重启丢失

AccessibilityManager 使用了 AppStorage 来管理状态，但没有正确初始化 PersistentStorage 的持久化键。

在 HarmonyOS 中：
- **AppStorage**: 仅内存存储，应用重启后数据丢失
- **PersistentStorage**: 持久化存储，需要通过 `PersistentStorage.persistProp()` 初始化键

## 解决方案

### 修复1：修复 getSettings() 和 update 方法

```typescript
// 修复后的 getSettings()
public getSettings(): AccessibilitySettings {
  const largeFont = AppStorage.get<boolean>(STORAGE_KEYS.LARGE_FONT);
  const highContrast = AppStorage.get<boolean>(STORAGE_KEYS.HIGH_CONTRAST);
  const vibration = AppStorage.get<boolean>(STORAGE_KEYS.VIBRATION);
  
  return {
    largeFontEnabled: largeFont !== undefined ? largeFont : false,
    highContrastEnabled: highContrast !== undefined ? highContrast : false,
    vibrationEnabled: vibration !== undefined ? vibration : true
  };
}

// 修复后的 updateLargeFont()
public updateLargeFont(enabled: boolean): void {
  console.info('大字体设置更新请求:', enabled);
  AppStorage.set<boolean>(STORAGE_KEYS.LARGE_FONT, enabled);
  
  // 立即构建设置对象，不依赖getSettings()
  const settings: AccessibilitySettings = {
    largeFontEnabled: enabled,  // 使用传入的值，不从AppStorage读取
    highContrastEnabled: AppStorage.get<boolean>(STORAGE_KEYS.HIGH_CONTRAST) ?? false,
    vibrationEnabled: AppStorage.get<boolean>(STORAGE_KEYS.VIBRATION) ?? true
  };
  
  this.debounceSave(settings);
  console.info('大字体设置已更新:', enabled);
}
```

关键改进：
1. 使用 `!== undefined` 而不是 `||` 来检查值是否存在
2. 在 update 方法中直接使用传入的 `enabled` 值，而不是从 AppStorage 重新读取
3. 使用 `??` 空值合并运算符代替 `||`

### 修复2：添加 PersistentStorage 键初始化

```typescript
private initPersistentStorageKeys(): void {
  try {
    // 为每个无障碍设置键初始化 PersistentStorage
    PersistentStorage.persistProp(STORAGE_KEYS.LARGE_FONT, DEFAULT_SETTINGS.largeFontEnabled);
    PersistentStorage.persistProp(STORAGE_KEYS.HIGH_CONTRAST, DEFAULT_SETTINGS.highContrastEnabled);
    PersistentStorage.persistProp(STORAGE_KEYS.VIBRATION, DEFAULT_SETTINGS.vibrationEnabled);
    console.info('PersistentStorage 键初始化成功');
  } catch (err) {
    console.warn('PersistentStorage 键初始化失败:', err);
  }
}

public async init(): Promise<void> {
  try {
    // 先初始化 PersistentStorage 的键（确保持久化）
    this.initPersistentStorageKeys();
    
    const settings = await this.loadSettings();
    this.syncToAppStorage(settings);
    console.info('无障碍设置初始化成功:', JSON.stringify(settings));
  } catch (err) {
    console.error('初始化无障碍设置失败:', err);
    this.syncToAppStorage(DEFAULT_SETTINGS);
  }
}
```

### 修复3：更新 syncToAppStorage 方法

```typescript
private syncToAppStorage(settings: AccessibilitySettings): void {
  // 使用 AppStorage.set 更新已存在的 PersistentStorage 键
  // 这样可以触发持久化保存
  AppStorage.set<boolean>(STORAGE_KEYS.LARGE_FONT, settings.largeFontEnabled);
  AppStorage.set<boolean>(STORAGE_KEYS.HIGH_CONTRAST, settings.highContrastEnabled);
  AppStorage.set<boolean>(STORAGE_KEYS.VIBRATION, settings.vibrationEnabled);
}
```

## 工作原理

### 设置更新流程

1. **用户点击Toggle**：
   - Toggle的 `onChange` 回调被触发
   - 调用 `accessibilityManager.updateLargeFont(true)`
   - `AppStorage.set()` 立即更新值
   - 构建包含新值的 settings 对象（不从 AppStorage 读取）
   - 防抖保存到 StorageUtil

2. **@StorageLink 同步**：
   - AppStorage 值改变后，所有使用 `@StorageLink` 的组件自动更新
   - Toggle 的 `isOn` 属性通过 `enabled` 参数获得新值
   - UI 立即反映新状态

3. **持久化保存**：
   - PersistentStorage 自动保存 AppStorage 的变化
   - StorageUtil 在 500ms 后保存完整的 JSON 设置

### 应用重启恢复流程

1. **应用启动**：
   - `EntryAbility.onCreate()` 调用 `AccessibilityManager.getInstance().init()`
   - `initPersistentStorageKeys()` 初始化持久化键
   - PersistentStorage 自动从持久化存储恢复值到 AppStorage

2. **加载设置**：
   - `loadSettings()` 从 StorageUtil 加载 JSON 设置
   - `syncToAppStorage()` 同步到 AppStorage（如果 JSON 存在）
   - 所有使用 `@StorageLink` 的组件自动获取持久化的值

## 双重持久化机制

现在系统使用双重持久化：

1. **PersistentStorage**: 持久化 AppStorage 中的三个独立键
   - 优点：自动同步，组件可以直接使用 `@StorageLink`
   - 用途：实时状态管理
   - 触发：AppStorage 值改变时自动保存

2. **StorageUtil (Preferences)**: 持久化完整的 JSON 设置对象
   - 优点：结构化存储，便于备份和恢复
   - 用途：设置的完整快照
   - 触发：防抖保存（500ms 延迟）

## 技术说明

### 为什么不能使用 `||` 运算符

```typescript
// 错误：无法区分 false 和 undefined
AppStorage.get<boolean>(key) || false

// 正确：明确检查 undefined
const value = AppStorage.get<boolean>(key);
value !== undefined ? value : false

// 或使用空值合并运算符
AppStorage.get<boolean>(key) ?? false
```

### 为什么要在 update 方法中直接使用传入值

```typescript
// 错误：可能读取到旧值或 undefined
public updateLargeFont(enabled: boolean): void {
  AppStorage.set(KEY, enabled);
  const settings = this.getSettings();  // 可能读取到旧值
  this.save(settings);
}

// 正确：直接使用传入的新值
public updateLargeFont(enabled: boolean): void {
  AppStorage.set(KEY, enabled);
  const settings = {
    largeFontEnabled: enabled,  // 使用传入值，确保正确
    ...
  };
  this.save(settings);
}
```

### PersistentStorage.persistProp() 的作用

```typescript
PersistentStorage.persistProp('key', defaultValue);
```

这个调用会：
1. 在 AppStorage 中创建一个键（如果不存在）
2. 将该键标记为持久化
3. 从持久化存储中恢复值（如果存在）
4. 当值改变时自动保存到持久化存储

## 注意事项

1. **初始化顺序很重要**：
   - 必须先调用 `initPersistentStorageKeys()`
   - 然后才能使用 `AppStorage.set()`

2. **预览器限制**：
   - 在 DevEco Studio 预览器中，PersistentStorage 可能不完全工作
   - 建议在真机或模拟器上测试持久化功能

3. **兼容性**：
   - 此修复与现有代码完全兼容
   - 不影响其他功能的使用

4. **性能考虑**：
   - PersistentStorage 自动保存是同步的，性能很好
   - StorageUtil 使用防抖保存，避免频繁写入
