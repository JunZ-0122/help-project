# 诊断志愿者聊天发送消息问题

## 快速诊断清单

### ✅ 第1步：检查后端数据

运行以下 SQL 查询，检查数据库中的数据：

```sql
-- 检查求助请求数据
SELECT 
    id,
    user_id,
    user_name,
    volunteer_id,
    volunteer_name,
    title,
    status
FROM help_requests
WHERE status IN ('assigned', 'in-progress')
LIMIT 5;
```

**预期结果**:
- `user_id` 字段不应该为 NULL
- `volunteer_id` 字段应该有值（已接单的请求）

**如果 user_id 为 NULL**:
```sql
-- 修复数据：将 user_id 设置为实际的用户ID
UPDATE help_requests 
SET user_id = 'user001' 
WHERE user_id IS NULL AND id = 'req001';
```

## 常见问题和解决方案

### 问题 1: targetUserId 为空

**症状**:
```
handleSend - Cannot send: text=true, targetUserId=false, isSending=false
```

**原因**:
- 后端 API 返回的 `userId` 为空
- 数据库中 `user_id` 字段为 NULL

**解决方案**:
1. 运行 SQL 查询检查数据：
   ```sql
   SELECT id, user_id, title FROM help_requests WHERE id = 'req001';
   ```

2. 如果 `user_id` 为 NULL，更新数据：
   ```sql
   UPDATE help_requests SET user_id = 'user001' WHERE id = 'req001';
   ```

3. 重新测试发送消息

### 问题 2: request.userId 为 null

**症状**:
```
resolveChatTarget - request.userId: null, request.volunteerId: user002
resolveChatTarget - Set target as help-seeker: 
```

**原因**:
- 后端 Mapper 查询没有包含 `user_id` 字段
- 数据库字段名不匹配

**解决方案**:
1. 检查 `HelpRequestMapper.xml`:
   ```xml
   <select id="selectById" resultType="com.csi.help.entity.HelpRequest">
       SELECT 
           id,
           user_id,        <!-- 确保包含这个字段 -->
           user_name,
           volunteer_id,
           volunteer_name,
           ...
       FROM help_requests
       WHERE id = #{id}
   </select>
   ```

2. 重启后端服务

3. 重新测试

### 问题 3: 角色判断错误

**症状**:
```
resolveChatTarget - currentUserRole: help-seeker, currentUserId: user002
```

但实际上 user002 是志愿者。

**原因**:
- 用户信息存储错误
- 登录时角色信息不正确

**解决方案**:
1. 检查登录响应：
   ```powershell
   $response = Invoke-RestMethod -Uri "http://localhost:8080/api/auth/login" -Method POST -ContentType "application/json" -Body $loginBody
   Write-Host "User role: $($response.data.user.role)"
   ```

2. 确认数据库中的用户角色：
   ```sql
   SELECT id, name, role FROM users WHERE phone = '13800138002';
   ```

3. 如果角色错误，更新数据：
   ```sql
   UPDATE users SET role = 'volunteer' WHERE phone = '13800138002';
   ```

## 测试脚本

创建一个完整的测试脚本 `test_volunteer_chat.ps1`:

```powershell
# 测试志愿者聊天功能
$baseUrl = "http://localhost:8080/api"

Write-Host "=== 测试志愿者聊天功能 ===" -ForegroundColor Cyan

# 1. 志愿者登录
Write-Host "`n步骤 1: 志愿者登录" -ForegroundColor Yellow
$loginBody = @{
    phone = "13800138002"
    password = "123456"
} | ConvertTo-Json

$loginResponse = Invoke-RestMethod -Uri "$baseUrl/auth/login" -Method POST -ContentType "application/json" -Body $loginBody
$token = $loginResponse.data.token
$volunteerId = $loginResponse.data.user.id
Write-Host "✓ 登录成功: $volunteerId" -ForegroundColor Green

# 2. 获取第一个求助请求
Write-Host "`n步骤 2: 获取求助请求列表" -ForegroundColor Yellow
$headers = @{ "Authorization" = "Bearer $token" }
$requestsUrl = "$baseUrl/help-requests" + "?page=1" + "&size=10"
$requestsResponse = Invoke-RestMethod -Uri $requestsUrl -Method GET -Headers $headers
$requestId = $requestsResponse.data.records[0].id
Write-Host "✓ 找到请求: $requestId" -ForegroundColor Green

# 3. 获取请求详情
Write-Host "`n步骤 3: 获取请求详情" -ForegroundColor Yellow
$requestDetail = Invoke-RestMethod -Uri "$baseUrl/help-requests/$requestId" -Method GET -Headers $headers
Write-Host "请求ID: $($requestDetail.data.id)" -ForegroundColor Gray
Write-Host "求助者ID: $($requestDetail.data.userId)" -ForegroundColor Gray
Write-Host "求助者姓名: $($requestDetail.data.userName)" -ForegroundColor Gray
Write-Host "志愿者ID: $($requestDetail.data.volunteerId)" -ForegroundColor Gray

if (-not $requestDetail.data.userId) {
    Write-Host "✗ 错误: userId 为空！" -ForegroundColor Red
    Write-Host "这就是为什么志愿者无法发送消息的原因。" -ForegroundColor Red
    exit 1
} else {
    Write-Host "✓ userId 正常: $($requestDetail.data.userId)" -ForegroundColor Green
}

# 4. 发送测试消息
Write-Host "`n步骤 4: 发送测试消息" -ForegroundColor Yellow
$messageBody = @{
    requestId = $requestId
    receiverId = $requestDetail.data.userId
    text = "测试消息 $(Get-Date -Format 'HH:mm:ss')"
    type = "text"
} | ConvertTo-Json

$headers = @{
    "Authorization" = "Bearer $token"
    "Content-Type" = "application/json"
}

try {
    $sendResponse = Invoke-RestMethod -Uri "$baseUrl/chat-messages/send" -Method POST -Headers $headers -Body $messageBody
    Write-Host "✓ 消息发送成功: $($sendResponse.data.id)" -ForegroundColor Green
    Write-Host "消息内容: $($sendResponse.data.text)" -ForegroundColor Gray
} catch {
    Write-Host "✗ 消息发送失败: $_" -ForegroundColor Red
}

Write-Host "`n=== 测试完成 ===" -ForegroundColor Cyan
```
