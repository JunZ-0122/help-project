# 数据库字段缺失修复指南

错误时间：2026-03-13 23:43
状态：⚠️ 需要修复

---

## 错误信息

```
Unknown column 'created_at' in 'field list'
```

志愿者接单时，后端尝试向 `volunteer_orders` 表插入数据，但数据库表中缺少 `created_at` 和 `updated_at` 字段。

---

## 根本原因

数据库表 `volunteer_orders` 的结构与代码中的定义不一致。可能的原因：

1. 数据库是用旧版本的 `schema.sql` 创建的
2. 表结构更新后，数据库没有同步更新
3. 手动创建表时遗漏了这两个字段

---

## 快速修复方案

### 方案一：执行更新脚本（推荐）

1. 打开 MySQL 命令行或工具（如 MySQL Workbench、Navicat）

2. 连接到数据库：
```bash
mysql -u root -p
```

3. 执行更新脚本：
```bash
mysql -u root -p help < src/main/resources/update-schema.sql
```

或者在 MySQL 命令行中：
```sql
USE help;
SOURCE C:/Users/22789/Desktop/endProject/help/src/main/resources/update-schema.sql;
```

### 方案二：手动添加字段

如果方案一不可行，可以手动执行以下 SQL：

```sql
USE help;

-- 添加 created_at 和 updated_at 字段
ALTER TABLE volunteer_orders 
ADD COLUMN created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间' AFTER images,
ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间' AFTER created_at;

-- 验证字段已添加
DESC volunteer_orders;
```

### 方案三：重建表（慎用）

如果表中没有重要数据，可以删除表并重新创建：

```sql
USE help;

-- 备份数据（如果有）
CREATE TABLE volunteer_orders_backup AS SELECT * FROM volunteer_orders;

-- 删除旧表
DROP TABLE volunteer_orders;

-- 重新创建表
```

## 常见问题

### Q1: 执行更新脚本时报错 "Table doesn't exist"

A: 表名可能不匹配。检查表名是单数还是复数：
```sql
SHOW TABLES LIKE '%volunteer%';
```

如果表名是 `volunteer_order`（单数），需要重命名：
```sql
RENAME TABLE volunteer_order TO volunteer_orders;
```

### Q2: 字段已存在，但仍然报错

A: 可能是字段类型或约束不匹配。删除字段后重新添加：
```sql
ALTER TABLE volunteer_orders DROP COLUMN created_at;
ALTER TABLE volunteer_orders DROP COLUMN updated_at;

ALTER TABLE volunteer_orders 
ADD COLUMN created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间';
```

### Q3: 后端仍然报错

A: 重启后端服务，确保连接到正确的数据库：
```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/help?useSSL=false&serverTimezone=UTC
    username: root
    password: 123456
```
