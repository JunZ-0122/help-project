# 志愿者聊天功能修复方案

## 问题诊断

您报告的问题：志愿者在聊天界面无法发送消息，日志显示：
```
handleSend - text: "1111", targetUserId: "", isSending: false
handleSend - Cannot send: text=true, targetUserId=false, isSending=false
```

**根本原因**：数据库中 `help_requests` 表的 `user_id` 字段为 NULL，导致前端无法获取聊天对象的用户ID。

## 完整的数据流程

1. 志愿者接单 → 创建 `volunteer_order`
2. 志愿者进入"帮扶进度"页面 (`ProgressPage`)
3. 点击"发送消息"按钮 → 跳转到 `ChatPage`，传递 `requestId` 参数
4. `ChatPage` 调用后端 API 获取求助请求详情
5. `resolveChatTarget()` 方法从 `request.userId` 提取目标用户ID
6. **问题点**：`request.userId` 为空，导致 `targetUserId` 为空字符串
7. 发送消息时检查失败，无法发送

## 修复方案

### 方案 1：修复数据库（推荐，立即生效）

**步骤 1**：检查当前数据库状态
```bash
cd help
mysql -u root -p -e "SELECT COUNT(*) as total, SUM(CASE WHEN user_id IS NULL OR user_id = '' THEN 1 ELSE 0 END) as null_count FROM help_requests" help
```

**步骤 2**：运行修复脚本
```bash
cd help
mysql -u root -p help < fix_chat_user_id.sql
```

**步骤 3**：验证修复结果
```bash
mysql -u root -p -e "SELECT id, user_id, user_phone, user_name FROM help_requests LIMIT 10" help
```

所有记录的 `user_id` 字段应该都有值。

**步骤 4**：重启后端服务
```bash
cd help
mvn spring-boot:run
```

**步骤 5**：测试聊天功能
1. 在 DevEco Studio 中重新运行前端应用
2. 登录志愿者账号 (13800138002 / 123456)
3. 进入一个活跃订单的"帮扶进度"页面
4. 点击"发送消息"
5. 输入消息并点击"发送"
6. 验证消息成功发送并显示在聊天记录中

2. DevEco Studio 的预览器使用的是旧的编译结果
3. 必须执行 Clean → Rebuild 才能看到新的日志
