# 角色选择功能修复完成

## 问题描述

用户在角色选择页面选择角色后，调用 `/api/users/me/role` 接口返回 500 错误，导致角色更新失败。

## 根本原因

后端 `UserMapper.xml` 的 `update` 方法中缺少 `role` 字段的更新语句，导致 MyBatis 无法更新用户的角色字段。

## 修复内容

### 1. 修复 UserMapper.xml

**文件**: `help/src/main/resources/mapper/UserMapper.xml`

在 `<update id="update">` 标签中添加了 role 字段的更新：

```xml
<update id="update">
    UPDATE users
    <set>
        <if test="name != null">name = #{name},</if>
        <if test="avatar != null">avatar = #{avatar},</if>
        <if test="role != null">role = #{role},</if>  <!-- 新增 -->
        <if test="age != null">age = #{age},</if>
        <!-- ... 其他字段 ... -->
        updated_at = NOW()
    </set>
    WHERE id = #{id}
</update>
```

### 2. 重启后端服务

由于 MyBatis XML 配置文件的修改需要重启服务才能生效，执行了以下操作：

1. 停止旧的后端进程（PID 2072）
2. 重新启动后端服务：`.\mvnw.cmd spring-boot:run`
3. 服务成功启动在端口 8080

## 技术要点

1. **MyBatis 动态 SQL**：使用 `<if test="...">` 标签实现字段的条件更新
2. **配置热加载**：MyBatis XML 配置修改后必须重启服务
3. **业务规则**：在 Controller 层验证用户是否已有角色，防止重复修改
4. **Token 认证**：使用 JWT Bearer Token 进行用户身份验证
