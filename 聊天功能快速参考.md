## 📱 用户操作

### 发送消息
```
1. 在输入框输入文字
2. 点击"发送"按钮或按回车键
3. 消息立即显示在聊天列表中
4. 输入框自动清空
```

### 切换输入方式
```
1. 点击"文字"按钮 - 使用键盘输入（默认）
2. 点击"手写"按钮 - 显示开发中提示
3. 点击"语音"按钮 - 显示开发中提示
```

### 查看连接状态
```
页面顶部显示：
- 已连接 (绿色) - WebSocket 正常连接
- 连接中 (黄色) - 正在建立连接
- 重连中 (黄色) - 连接断开，正在重连
- 连接异常 (红色) - 连接失败
- 已断开 (灰色) - 未连接
```

## 🔧 技术实现

### 前端 (ArkTS)
```typescript
// 发送消息
MessageApi.sendMessage({
  requestId: this.requestId,
  receiverId: this.targetUserId,
  text: text,
  type: 'text'
})

// 接收实时消息
this.webSocketService.onMessage(this.requestId, (message) => {
  this.handleRealtimeMessage(message);
})

// 连接 WebSocket
await this.webSocketService.connect(token);
```

### 后端 (Spring Boot)
```java
// WebSocket 端点
ws://localhost:8080/ws/chat?token=<JWT_TOKEN>

// 推送消息
chatWebSocketHandler.sendChatMessage(message);

// 推送已读回执
chatWebSocketHandler.sendReadReceipt(userId, event);
```

## 🐛 常见问题

### Q: 发送按钮点不了？
A: 检查输入框是否有内容，只有输入内容后按钮才会启用。

### Q: 对方收不到消息？
A: 检查 WebSocket 连接状态，确保双方都显示"已连接"。

### Q: 消息重复显示？
A: 这是一个已知问题，刷新页面可以解决。代码中有去重逻辑，正常情况不会重复。

### Q: 手写/语音输入不能用？
A: 这两个功能正在开发中，点击会显示提示对话框。

### Q: 连接状态一直显示"重连中"？
A: 检查后端服务是否正常运行，检查网络连接是否正常。
