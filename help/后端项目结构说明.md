## 🏗️ 三层架构说明

### 1. Controller 层（控制器层）
**职责**: 接收 HTTP 请求，调用 Service 层处理业务逻辑，返回响应

**特点**:
- 使用 `@RestController` 注解
- 使用 `@RequestMapping` 定义路由
- 参数验证使用 `@Validated`
- 统一返回 `Result<T>` 格式

**示例**:
```java
@RestController
@RequestMapping("/api/auth")
public class AuthController {
    @PostMapping("/login")
    public Result<LoginResponse> login(@RequestBody LoginRequest request) {
        // 调用 Service 层
        LoginResponse response = authService.login(request);
        return Result.success(response);
    }
}
```

### 2. Service 层（服务层）
**职责**: 处理业务逻辑，调用 Mapper 层访问数据库

**特点**:
- 使用 `@Service` 注解
- 使用 `@RequiredArgsConstructor` 注入依赖
- 包含业务逻辑和数据处理
- 事务管理

**示例**:
```java
@Service
@RequiredArgsConstructor
public class AuthService {
    private final UserMapper userMapper;
    
    public LoginResponse login(LoginRequest request) {
        // 业务逻辑
        User user = userMapper.findByPhone(request.getPhone());
        // ...
        return response;
    }
}
```

### 3. Mapper 层（数据访问层）
**职责**: 数据库操作，使用 MyBatis 进行 SQL 映射

**特点**:
- 使用 `@Mapper` 注解
- 接口方法对应 XML 中的 SQL
- 参数使用 `@Param` 注解

**示例**:
```java
@Mapper
public interface UserMapper {
    User findByPhone(@Param("phone") String phone);
    int insert(User user);
}
```

## 📦 核心组件说明

### 1. 统一响应格式 (Result)
```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": "2024-01-01T00:00:00"
}
```

### 2. 分页响应格式 (PageResult)
```json
{
  "items": [...],
  "total": 100,
  "page": 1,
  "pageSize": 10,
  "totalPages": 10
}
```

### 3. JWT 认证流程
```
1. 用户登录 → 生成 Token
2. 请求携带 Token → AuthInterceptor 验证
3. 验证通过 → 将用户信息存入 Request Attribute
4. Controller 获取用户信息 → 处理业务
```

### 4. 拦截器配置
- 所有 `/api/**` 路径需要认证
- 排除路径：登录、注册、发送验证码、刷新Token

## 📝 开发规范

### 1. 命名规范
- **类名**: 大驼峰，如 `UserService`
- **方法名**: 小驼峰，如 `findByPhone`
- **变量名**: 小驼峰，如 `userId`
- **常量名**: 全大写下划线，如 `TOKEN_EXPIRATION`

### 2. 包结构规范
- `controller`: 控制器
- `service`: 服务
- `mapper`: 数据访问
- `entity`: 实体类
- `dto`: 数据传输对象
- `common`: 通用类
- `config`: 配置类
- `util`: 工具类
- `interceptor`: 拦截器

### 3. 注释规范
- 类注释：说明类的职责
- 方法注释：说明方法的功能、参数、返回值
- 复杂逻辑：添加行内注释

## 🔍 常见问题

### Q1: 如何添加新的接口？
1. 在 `entity` 包创建实体类
2. 在 `mapper` 包创建 Mapper 接口
3. 在 `resources/mapper` 创建 XML 映射文件
4. 在 `service` 包创建 Service 类
5. 在 `controller` 包创建 Controller 类

### Q2: 如何处理跨域？
已在 `WebConfig` 中配置 CORS，允许所有来源

### Q3: 如何获取当前登录用户？
在 Controller 方法中使用 `@RequestAttribute`:
```java
public Result<T> method(@RequestAttribute String userId) {
    // userId 由 AuthInterceptor 注入
}
```

### Q4: 如何处理异常？
建议创建全局异常处理器 `@ControllerAdvice`
