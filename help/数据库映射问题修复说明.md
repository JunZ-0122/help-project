# 数据库映射问题修复说明

## 问题描述

后端的 Mapper XML 文件与数据库表结构不匹配，导致字段映射错误。

## 主要问题

### 1. 表名不一致
**问题**：Mapper XML 中使用的表名与实际数据库表名不一致

| Mapper XML 中的表名 | 实际数据库表名 | 状态 |
|-------------------|--------------|------|
| `volunteer_order` | `volunteer_orders` | ❌ 已修复 |
| `chat_message` | `chat_messages` | ❌ 已修复 |
| `review` | `reviews` | ❌ 已修复 |

### 2. 字段缺失
**问题**：实体类和 Mapper XML 缺少数据库表中的字段

#### VolunteerOrder 实体
- ❌ 缺少 `created_at` 字段 → ✅ 已添加
- ❌ 缺少 `updated_at` 字段 → ✅ 已添加
- ❌ 缺少大量业务字段（volunteer_name, help_seeker_id 等）→ ✅ 已添加

#### ChatMessage 实体
- ❌ Mapper XML 使用 `content` 字段，但数据库是 `text` → ✅ 已修复
- ❌ 缺少 `sender_name`, `sender_avatar` 等字段 → ✅ 已添加

#### Review 实体
- ❌ 实体类有 `updated_at` 但数据库没有 → ✅ 已移除
- ❌ 缺少 `reviewer_name`, `reviewee_name` 等字段 → ✅ 已添加

### 3. ResultMap 映射不完整
**问题**：Mapper XML 的 ResultMap 只映射了部分字段

所有 Mapper XML 的 ResultMap 已更新为完整映射。

## 如何应用修复

### 方案 1：重新创建数据库（推荐）

如果数据库中没有重要数据，建议删除并重新创建：

```bash
# 1. 删除旧数据库
mysql -u root -p -e "DROP DATABASE IF EXISTS help;"

# 2. 重新创建数据库和表
mysql -u root -p < src/main/resources/schema.sql

# 3. 导入测试数据
mysql -u root -p help < src/main/resources/test-data.sql
```

### 方案 2：更新现有数据库

如果数据库中有数据需要保留：

```bash
# 执行更新脚本
mysql -u root -p < src/main/resources/update-schema.sql
```

**注意**：如果表名不正确（单数形式），需要手动重命名：

```sql
USE help;

-- 重命名表（如果需要）
RENAME TABLE volunteer_order TO volunteer_orders;
RENAME TABLE chat_message TO chat_messages;
RENAME TABLE review TO reviews;
```

## 常见错误及解决方案

### 错误 1：Table 'help.volunteer_order' doesn't exist

**原因**：表名不匹配

**解决方案**：
```sql
RENAME TABLE volunteer_order TO volunteer_orders;
```

### 错误 2：Unknown column 'created_at' in 'field list'

**原因**：表中缺少字段

**解决方案**：
```sql
ALTER TABLE volunteer_orders 
ADD COLUMN created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
```

### 错误 3：Column 'volunteer_name' cannot be null

**原因**：Service 层没有设置必需字段

**解决方案**：已在 Service 层修复，确保使用最新代码
