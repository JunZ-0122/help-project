# 注解问题修复说明

## 问题描述

在 IntelliJ IDEA 中，`LoginRequest.java` 和 `RegisterRequest.java` 文件中的 `@NotBlank` 注解报错：
```
Cannot resolve symbol 'NotBlank'
```

## 问题原因

Spring Boot 3.x 使用的是 **Jakarta EE** 规范，而不是旧的 **Java EE (javax)** 规范。

在 Spring Boot 2.x 中，validation 注解的包名是：
```java
import javax.validation.constraints.NotBlank;
```

在 Spring Boot 3.x 中，validation 注解的包名改为：
```java
import jakarta.validation.constraints.NotBlank;
```

## 修复方案

### 已修复的文件

1. **LoginRequest.java**
   - ❌ 旧的导入：`import javax.validation.constraints.NotBlank;`
   - ✅ 新的导入：`import jakarta.validation.constraints.NotBlank;`

2. **RegisterRequest.java**
   - ❌ 旧的导入：`import javax.validation.constraints.NotBlank;`
   - ✅ 新的导入：`import jakarta.validation.constraints.NotBlank;`

### 依赖配置

`pom.xml` 中已经包含了正确的依赖：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

这个依赖在 Spring Boot 3.x 中会自动引入 Jakarta Validation API。

## 验证修复

### 1. 刷新 Maven 项目

在 IntelliJ IDEA 中：
1. 右键点击 `pom.xml`
2. 选择 `Maven` → `Reload Project`

或者使用快捷键：`Ctrl + Shift + O` (Windows/Linux) 或 `Cmd + Shift + I` (Mac)

### 2. 重新构建项目

```bash
mvn clean install
```

### 3. 检查错误是否消失

打开 `LoginRequest.java` 和 `RegisterRequest.java`，确认 `@NotBlank` 注解不再报错。

## Spring Boot 2.x 到 3.x 的其他变更

如果你的项目从 Spring Boot 2.x 升级到 3.x，还需要注意以下变更：

### 1. Servlet API
- ❌ `javax.servlet.*` 
- ✅ `jakarta.servlet.*`

### 2. Persistence API (JPA)
- ❌ `javax.persistence.*`
- ✅ `jakarta.persistence.*`

### 3. Validation API
- ❌ `javax.validation.*`
- ✅ `jakarta.validation.*`

### 4. Transaction API
- ❌ `javax.transaction.*`
- ✅ `jakarta.transaction.*`

## 常见问题

### Q1: 为什么要从 javax 改为 jakarta？

**A**: Oracle 将 Java EE 捐赠给 Eclipse 基金会后，由于商标问题，包名从 `javax.*` 改为 `jakarta.*`。Spring Boot 3.x 采用了新的 Jakarta EE 规范。

### Q2: 如何批量替换导入？

**A**: 在 IntelliJ IDEA 中：
1. 按 `Ctrl + Shift + R` (Windows/Linux) 或 `Cmd + Shift + R` (Mac)
2. 在 "Find" 中输入：`import javax.validation`
3. 在 "Replace" 中输入：`import jakarta.validation`
4. 选择 "In Project" 范围
5. 点击 "Replace All"

### Q3: Maven 依赖冲突怎么办？

**A**: 如果遇到依赖冲突，可以使用以下命令查看依赖树：
```bash
mvn dependency:tree
```

确保没有同时引入 `javax.validation` 和 `jakarta.validation` 的依赖。

## 总结

- ✅ 已将所有 `javax.validation` 导入更新为 `jakarta.validation`
- ✅ `pom.xml` 中的依赖配置正确
- ✅ 项目可以正常编译和运行

如果还有其他注解报错，请检查是否也需要从 `javax.*` 更新为 `jakarta.*`。
