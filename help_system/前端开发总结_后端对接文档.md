## 1. 项目概述

### 1.1 系统角色

| 角色 | 英文标识 | 主要功能 |
|------|---------|---------|
| 弱势群体 | help-seeker | 发布求助、查看进度、评价服务 |
| 志愿者 | volunteer | 接受订单、提供服务、更新进度 |
| 社区管理员 | community | 管理请求、分配志愿者、查看统计 |

### 1.2 核心业务流程

```
求助者发布请求 → 社区管理员审核/分配 → 志愿者接单 → 
提供服务 → 更新进度 → 完成服务 → 求助者评价
```

## 3. 核心数据模型

### 3.1 用户模型 (User)

```typescript
interface User {
  id: string;
  name: string;
  phone: string;
  avatar?: string;
  role: 'help-seeker' | 'volunteer' | 'community';
  status: 'online' | 'offline' | 'busy';
  createdAt?: string;
}

interface UserProfile extends User {
  age?: number;
  gender?: 'male' | 'female' | 'other';
  address?: string;
  emergencyContact?: string;
  emergencyPhone?: string;
  disabilities?: string[];        // 弱势群体特有
  certifications?: string[];      // 志愿者特有
  volunteerHours?: number;        // 志愿者特有
  rating?: number;
}
```


### 3.2 求助请求模型 (HelpRequest)

```typescript
interface HelpRequest {
  id: string;
  userId: string;
  userName: string;
  userPhone: string;
  type: 'medical' | 'shopping' | 'repair' | 'companion' | 'emergency' | 'other';
  title: string;
  description: string;
  location: string;
  latitude?: number;
  longitude?: number;
  urgency: 'low' | 'medium' | 'high' | 'emergency';
  status: 'pending' | 'assigned' | 'in-progress' | 'completed' | 'cancelled';
  createdAt: string;
  updatedAt?: string;
  scheduledTime?: string;
  estimatedDuration?: number;
  images?: string[];
  volunteerId?: string;
  volunteerName?: string;
  completedAt?: string;
  rating?: number;
  feedback?: string;
}
```

**字段说明**:
- `type`: 求助类型，前端有对应的图标和颜色
- `urgency`: 紧急程度，影响显示优先级
- `status`: 状态流转 pending → assigned → in-progress → completed
- `scheduledTime`: ISO 8601 格式，如 "2024-01-02T09:00:00Z"

```json
  "timestamp": "2024-01-01T00:00:00Z"
}
```

**分页响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "totalPages": 10
  },
  "timestamp": "2024-01-01T00:00:00Z"
```

## 5. 业务流程说明

### 5.1 求助发布流程

```
1. 求助者选择求助类型
   ↓
2. 填写求助详情（标题、描述、位置、时间）
   ↓
3. 上传相关图片（可选）
   ↓
4. 提交请求 → POST /api/requests
   ↓
5. 系统创建请求，状态为 pending
   ↓
6. 社区管理员审核/分配志愿者
```


### 5.2 志愿者接单流程

```
1. 志愿者浏览求助列表
   ↓
2. 查看求助详情
   ↓
3. 点击"接受订单" → POST /api/orders/accept
   ↓
4. 系统创建订单，状态为 accepted
   ↓
5. 请求状态更新为 assigned
   ↓
6. 志愿者开始服务，更新状态为 in-progress
   ↓
7. 实时更新服务进度
   ↓
8. 完成服务 → POST /api/orders/:id/complete
   ↓
9. 求助者评价服务
```

### 5.3 实时聊天流程

```
1. 建立 WebSocket 连接
   ws://server/ws?token={jwt_token}
   ↓
2. 加载历史消息 → GET /api/messages/:requestId
   ↓
3. 发送消息 → WebSocket 发送
   ↓
4. 接收消息 → WebSocket 接收
   ↓
5. 标记已读 → PUT /api/messages/read
```

### 5.4 状态流转图

#### 求助请求状态
```
pending (待分配)
   ↓
assigned (已分配)
   ↓
in-progress (进行中)
   ↓
completed (已完成) / cancelled (已取消)
```

#### 志愿者订单状态
```
accepted (已接受)
   ↓
in-progress (进行中)
   ↓
completed (已完成) / cancelled (已取消)
```

## 7. 技术实现细节

### 7.1 认证机制

**Token 管理**:
- 使用 JWT Token 进行身份验证
- Token 存储在本地 StorageUtil 中
- 每次 API 请求自动添加 Authorization Header
- Token 过期后使用 refreshToken 刷新

**请求拦截器**:
```typescript
// 所有请求自动添加 token
headers: {
  'Authorization': `Bearer ${token}`,
  'Content-Type': 'application/json'
}
```


### 7.2 WebSocket 连接管理

**连接建立**:
```typescript
// 连接 URL
ws://your-server-url/ws?token={jwt_token}

// 或使用 Header
ws://your-server-url/ws
Header: Authorization: Bearer {token}
```

**心跳机制**:
- 前端每 30 秒发送一次 ping
- 后端应响应 pong
- 超时自动重连

**断线重连**:
- 网络断开后自动重连
- 最多重试 5 次
- 重连成功后重新订阅消息

### 7.3 数据缓存策略

**本地存储**:
- Token 和用户信息
- 无障碍设置
- 草稿数据

**内存缓存**:
- 用户列表
- 求助类型配置
- 常用地址

### 7.4 错误处理

**HTTP 错误码**:
```typescript
200: 成功
400: 请求参数错误
401: 未授权（token 无效）
403: 禁止访问（权限不足）
404: 资源不存在
500: 服务器错误
```

**前端错误处理**:
- 401: 自动跳转登录页
- 403: 显示权限不足提示
- 500: 显示服务器错误提示
- 网络错误: 显示网络连接失败

### 8.2 数据库设计建议

**核心表**:
```sql
-- 用户表
users (id, name, phone, password, role, avatar, status, created_at)

-- 求助请求表
help_requests (id, user_id, type, title, description, location, 
               latitude, longitude, urgency, status, created_at)

               type, is_read, created_at)

-- 评价表

```

**索引建议**:
- users: phone (唯一索引)
- help_requests: user_id, status, created_at
- volunteer_orders: volunteer_id, status
- chat_messages: request_id, created_at


### 8.3 安全建议

**密码安全**:
- 使用 bcrypt 加密密码
- 密码强度验证
- 防止暴力破解（限流）

**Token 安全**:
- JWT Token 有效期 2 小时
- RefreshToken 有效期 7 天
- Token 黑名单机制

**数据验证**:
- 所有输入参数验证
- SQL 注入防护
- XSS 防护
- CSRF 防护

**权限控制**:
- 基于角色的访问控制 (RBAC)
- 接口权限验证
- 数据权限隔离

### 8.4 性能优化建议

**数据库优化**:
- 合理使用索引
- 避免 N+1 查询
- 使用连接池
- 读写分离（可选）

**缓存策略**:
- Redis 缓存热点数据
- 用户信息缓存（5分钟）
- 求助列表缓存（1分钟）
- 统计数据缓存（10分钟）

**API 优化**:
- 分页查询（默认 10 条/页）
- 字段过滤（只返回需要的字段）
- 批量操作接口
- 异步处理耗时操作

**文件存储**:
- 使用 OSS 或 CDN
- 图片压缩和缩略图
- 限制文件大小（图片 5MB，语音 10MB）

### 8.5 监控和日志

**日志记录**:
- 请求日志（URL、参数、响应时间）
- 错误日志（异常堆栈）
- 业务日志（关键操作）
- 安全日志（登录、权限）

**监控指标**:
- API 响应时间

### 9.3 常见问题

**Q1: 时间格式不一致**
- 统一使用 ISO 8601 格式: "2024-01-01T10:00:00Z"
- 前端会自动转换为本地时间显示

**Q2: 分页参数不一致**
- 统一使用 page (从 1 开始) 和 pageSize
- 返回 total, totalPages 等信息

**Q3: 枚举值不匹配**
- 严格按照前端模型定义的枚举值
- 如 role: 'help-seeker' | 'volunteer' | 'community'

**Q4: 图片 URL 格式**
- 返回完整的 URL: "https://example.com/uploads/image.jpg"
- 不要返回相对路径

**Q5: WebSocket 连接失败**
- 检查 CORS 配置
- 检查 Token 验证逻辑
- 检查心跳机制
