# 应用重新编译指南

## 问题现象

修改了代码但应用仍然使用旧代码，错误依然存在：
```
Error: Internal error. UI execution context not found.
```

## 原因分析

HarmonyOS 应用的热重载（Hot Reload）有时不会完全更新代码，特别是：
1. 修改了异步逻辑
2. 修改了路由相关代码
3. 修改了生命周期方法

## 解决方案

### 方案 1: 完全重新编译（推荐）

#### 步骤 1: 停止当前运行
在 DevEco Studio 中：
1. 点击工具栏的 Stop 按钮（红色方块）
2. 或按快捷键 `Ctrl+F2`

#### 步骤 2: 清理构建缓存
```powershell
# 在 help_system 目录下执行
Remove-Item -Recurse -Force .hvigor\cache
Remove-Item -Recurse -Force entry\.preview
Remove-Item -Recurse -Force entry\build
```

或在 DevEco Studio 中：
1. 菜单：Build → Clean Project
2. 等待清理完成

#### 步骤 3: 重新构建
1. 菜单：Build → Rebuild Project
2. 等待构建完成（可能需要 1-2 分钟）

#### 步骤 4: 重新运行
1. 点击 Run 按钮（绿色三角形）
2. 或按快捷键 `Shift+F10`

### 方案 2: 使用命令行重新编译

```powershell
# 在 help_system 目录下执行

# 1. 清理缓存
Remove-Item -Recurse -Force .hvigor\cache -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force entry\.preview -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force entry\build -ErrorAction SilentlyContinue

# 2. 重新构建
hvigorw clean
hvigorw assembleHap

# 3. 在 DevEco Studio 中重新运行
```

### 方案 3: 卸载并重新安装应用

如果上述方法都不行，可以完全卸载应用：

#### 在模拟器/真机上：
1. 长按应用图标
2. 选择"卸载"
3. 在 DevEco Studio 中重新运行

#### 或使用命令：
```bash
# 查找应用包名
hdc shell bm dump -a

# 卸载应用
hdc uninstall com.csi.help

# 重新安装
hvigorw assembleHap
hdc install entry/build/default/outputs/default/entry-default-signed.hap
```

## 验证修改是否生效

### 检查日志
运行应用后，查看日志中是否有新的输出：

```
✅ 应该看到: "使用 setTimeout 确保在 UI 上下文中执行路由跳转"
✅ 应该看到: "页面跳转成功"
❌ 不应该看到: "UI execution context not found"
```

### 检查行为
1. 输入手机号和验证码
2. 点击登录
3. 应该看到"登录成功"提示
4. 100ms 后应该自动跳转到角色选择页面

## 常见问题

### Q1: 清理缓存后构建很慢
A: 第一次清理后重新构建需要重新编译所有文件，这是正常的。后续的增量构建会快很多。

### Q2: 构建失败
A: 检查是否有编译错误：
```powershell
# 查看详细错误信息
hvigorw assembleHap --stacktrace
```

### Q3: 应用无法启动
A: 检查模拟器/真机是否正常运行：
```bash
# 查看设备列表
hdc list targets

# 查看设备日志
hdc shell hilog
```

### Q4: 修改仍然不生效
A: 确认修改的文件路径是否正确：
```
help_system/entry/src/main/ets/pages/LoginPage.ets
```

## 开发建议

### 1. 使用增量构建
日常开发时，小的修改可以使用热重载：
- 修改 UI 样式
- 修改文本内容
- 修改简单的逻辑

### 2. 完全重新构建的场景
以下情况建议完全重新构建：
- 修改了路由配置
- 修改了异步逻辑
- 修改了生命周期方法
- 添加了新的依赖
- 修改了配置文件

### 3. 定期清理缓存
建议每天开始开发前清理一次缓存，避免累积问题。

## 快速命令

### Windows PowerShell
```powershell
# 一键清理并重新构建
cd help_system
Remove-Item -Recurse -Force .hvigor\cache -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force entry\.preview -ErrorAction SilentlyContinue
hvigorw clean
hvigorw assembleHap
```

### 创建快捷脚本
创建 `rebuild.ps1` 文件：
```powershell
# rebuild.ps1
Write-Host "清理缓存..." -ForegroundColor Yellow
Remove-Item -Recurse -Force .hvigor\cache -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force entry\.preview -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force entry\build -ErrorAction SilentlyContinue

Write-Host "重新构建..." -ForegroundColor Yellow
hvigorw clean
hvigorw assembleHap

Write-Host "完成！请在 DevEco Studio 中重新运行应用。" -ForegroundColor Green
```

使用方法：
```powershell
cd help_system
.\rebuild.ps1
```

## 总结

修改代码后如果不生效，按以下顺序尝试：
1. ✅ 停止应用
2. ✅ 清理缓存
3. ✅ 重新构建
4. ✅ 重新运行
5. ✅ 验证日志

如果还是不行，完全卸载应用后重新安装。
