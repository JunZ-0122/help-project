# Help System - 前后端联调指南

## 📋 联调准备

### 1. 后端准备

#### 启动后端服务
```bash
cd help
mvn spring-boot:run
```

确保后端服务运行在 `http://localhost:8080`

#### 验证后端服务
```bash
# 测试登录接口
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138001","password":"123456"}'
```

### 2. 前端准备

#### 配置 API 地址
前端 API 地址已配置在 `entry/src/main/ets/services/ApiService.ets`：
```typescript
const API_BASE_URL = 'http://localhost:8080/api';
```

**注意**：如果使用真机测试，需要将 `localhost` 改为电脑的局域网 IP 地址。

#### 获取电脑 IP 地址
```bash
# Windows
ipconfig

# 查找 IPv4 地址，例如：192.168.1.100
```

然后修改 `ApiService.ets`：
```typescript
const API_BASE_URL = 'http://192.168.1.100:8080/api';
```

## 🔍 调试技巧

### 1. 查看前端日志
在 DevEco Studio 中查看 Log 输出：
```
开始登录流程
验证通过，开始调用登录 API
登录成功: {"token":"...","user":{...}}
```

### 2. 查看后端日志
在后端控制台查看请求日志：
```
2024-03-07 10:00:00.000  INFO --- [http-nio-8080-exec-1] c.c.h.c.AuthController : 用户登录: 13800138001
```

### 3. 使用网络抓包
在 DevEco Studio 中使用 Network Profiler 查看 HTTP 请求：
- 请求 URL
- 请求 Headers
- 请求 Body
- 响应数据

### 4. 常见问题排查

#### 问题 1：网络请求失败
**症状**：提示"登录失败，请重试"

**排查步骤**：
1. 检查后端服务是否启动
2. 检查 API 地址是否正确
3. 检查网络连接（真机需要在同一局域网）
4. 查看前端日志中的错误信息

#### 问题 2：跨域问题
**症状**：请求被拒绝，CORS 错误

**解决方案**：
后端已配置 CORS，允许所有来源。如果仍有问题，检查 `WebConfig.java`。

#### 问题 3：Token 未保存
**症状**：登录成功但后续请求未携带 Token

**排查步骤**：
1. 检查 `StorageUtil.set('token', ...)` 是否执行
2. 检查 `StorageUtil.get('token')` 是否能获取到值
3. 查看 `HttpClient.getHeaders()` 是否正确添加 Authorization Header
