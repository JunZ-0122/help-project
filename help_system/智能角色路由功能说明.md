# 智能角色路由功能说明

## 功能概述

实现了基于用户角色的智能路由跳转功能，提升用户体验：

1. **新用户**（首次登录，role 为 null）→ 跳转到角色选择页
2. **已有角色用户**（role 已设置）→ 直接跳转到对应角色主页

## 实现逻辑

### 1. 后端修改

#### 1.1 登录自动注册
修改 `AuthService.login()` 方法，支持验证码登录自动注册：

```java
// 如果用户不存在且是验证码登录，自动创建用户
if (user == null && "sms".equals(request.getLoginType())) {
    user = new User();
    user.setId(IdUtil.simpleUUID());
    user.setName("用户" + request.getPhone().substring(7));
    user.setPhone(request.getPhone());
    user.setPassword(SecureUtil.md5(request.getPassword()));
    user.setRole(null); // 新用户角色为空
    user.setStatus("online");
    userMapper.insert(user);
}
```

#### 1.2 更新用户角色 API
新增 `PUT /api/users/me/role` 接口：

**请求体**:
```json
{
  "role": "help-seeker" | "volunteer" | "community"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "user001",
    "name": "张三",
    "phone": "13800138001",
    "role": "help-seeker",
    ...
  }
}
```

**限制**:
- 只能设置一次角色
- 已有角色的用户不允许修改

### 2. 前端修改

#### 2.1 LoginPage 智能路由
修改 `onLoginSuccess()` 方法，根据用户角色决定跳转：

```typescript
onLoginSuccess() {
  const user = this.loginUserData.user;
  const role = user.role;
  
  let targetUrl = '';
  let message = '';
  
  if (!role || role === '') {
    // 新用户 -> 角色选择页
    targetUrl = 'pages/RoleSelectPage';
    message = '欢迎！请选择您的身份';
  } else {
    // 已有角色 -> 对应主页
    switch (role) {
      case 'help-seeker':
        targetUrl = 'pages/help-seeker/HelpSeekerHomePage';
        message = '欢迎回来，求助者';
        break;
      case 'volunteer':
        targetUrl = 'pages/volunteer/VolunteerHomePage';
        message = '欢迎回来，志愿者';
        break;
      case 'community':
        targetUrl = 'pages/community/CommunityHomePage';
        message = '欢迎回来，管理员';
        break;
    }
  }
  
  router.pushUrl({ url: targetUrl, params: { user } });
}
```

#### 2.2 RoleSelectPage 更新角色
修改 `handleRoleSelect()` 方法，调用 API 更新角色：

```typescript
async handleRoleSelect(roleId: string, path: string, title: string) {
  try {
    // 调用后端 API 更新角色
    const updatedUser = await UserApi.updateUserRole(roleId);
    
    // 更新本地存储
    await StorageUtil.set('user', updatedUser);
    
    // 显示提示
    promptAction.showToast({ message: `已选择：${title}` });
    
    // 跳转到对应主页
    this.targetPath = path;
    this.roleUpdateSuccess = true; // 触发 @Watch 回调
  } catch (err) {
    promptAction.showToast({ message: '更新角色失败' });
  }
}
```

## 用户流程

### 场景 1：新用户首次登录

```
1. 用户输入手机号和验证码
   ↓
2. 点击"登录/注册"按钮
   ↓
3. 后端检测用户不存在，自动创建用户（role = null）
   ↓
4. 返回 Token 和用户信息
   ↓
5. 前端检测 role 为 null
   ↓
6. 跳转到角色选择页
   ↓
7. 用户选择角色（求助者/志愿者/社区管理）
   ↓
8. 调用 API 更新角色
   ↓
9. 跳转到对应角色主页
```

### 场景 2：已有角色用户登录

```
1. 用户输入手机号和验证码
   ↓
2. 点击"登录/注册"按钮
   ↓
3. 后端验证用户，返回用户信息（role = "help-seeker"）
   ↓
4. 前端检测 role 已设置
   ↓
5. 直接跳转到求助者主页
   ↓
6. 用户开始使用应用
```

## API 接口

### 1. 登录接口（支持自动注册）
**POST** `/api/auth/login`

**请求体**:
```json
{
  "phone": "13900000001",
  "password": "123456",
  "loginType": "sms"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGc...",
    "refreshToken": "eyJhbGc...",
    "expiresIn": 7200,
    "user": {
      "id": "user123",
      "name": "用户0001",
      "phone": "13900000001",
      "role": null,  // 新用户角色为 null
      "status": "online"
    }
  }
}
```

### 2. 更新角色接口
**PUT** `/api/users/me/role`

**请求头**:
```
Authorization: Bearer {token}
```

**请求体**:
```json
{
  "role": "help-seeker"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "user123",
    "name": "用户0001",
    "phone": "13900000001",
    "role": "help-seeker",  // 角色已更新
    "status": "online"
    }
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "用户已有角色，不允许修改",
  "data": null
}
```

## 注意事项

1. **角色只能设置一次**：用户选择角色后不能修改，确保数据一致性
2. **验证码登录自动注册**：使用验证码登录时，如果用户不存在会自动创建
3. **密码登录不自动注册**：使用密码登录时，如果用户不存在会返回错误
4. **Token 包含角色信息**：JWT Token 中包含用户角色，用于后续权限验证
5. **本地存储同步**：选择角色后需要更新本地存储的用户信息
