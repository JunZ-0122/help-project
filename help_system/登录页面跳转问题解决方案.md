# 登录页面跳转问题 - 最终解决方案

## 问题描述

在 HarmonyOS ArkTS 应用中，登录成功后尝试跳转到角色选择页时，出现以下错误：

```
Error: Internal error. UI execution context not found.
```

## 问题原因

Promise 异步回调可能不在 UI 线程中执行，导致：
- `router.pushUrl()` 无法找到 UI 执行上下文
- `promptAction.showToast()` 无法显示提示

## 最终解决方案：使用 @Watch 装饰器

### 核心思路

利用 ArkTS 的 `@Watch` 装饰器监听状态变化，在状态变化的回调中执行路由跳转。ArkUI 框架保证 `@Watch` 回调在 UI 上下文中执行。

### 实现步骤

#### 1. 定义状态变量
```typescript
@State @Watch('onLoginSuccess') loginSuccessful: boolean = false;
@State loginUserData: LoginResponse | null = null;
```

#### 2. 定义监听回调
```typescript
onLoginSuccess() {
  if (this.loginSuccessful && this.loginUserData) {
    console.info('检测到登录成功，准备跳转到角色选择页');
    
    // 显示成功提示
    promptAction.showToast({
      message: '登录成功',
      duration: 2000
    });
    
    // 执行路由跳转
    router.pushUrl({
      url: 'pages/RoleSelectPage',
      params: {
        user: this.loginUserData.user
      }
    }).then(() => {
      console.info('页面跳转成功');
    }).catch((err: Error) => {
      console.error('导航失败:', err);
      promptAction.showToast({
        message: '页面跳转失败: ' + err.message,
        duration: 2000
      });
    });
  }
}
```

#### 3. 在 Promise 回调中触发状态变化
```typescript
handleLogin() {
  this.isLoading = true;
  
  AuthService.login(loginRequest).then((response) => {
    console.info('登录成功:', JSON.stringify(response));
    console.info('Token 已保存，用户信息已保存');
    
    this.isLoading = false;
    
    // 保存登录数据并触发状态变化
    this.loginUserData = response;
    this.loginSuccessful = true; // 这会触发 @Watch 回调
    
    console.info('登录流程完成，等待状态监听器触发跳转');
  }).catch((err: Error) => {
    console.error('登录失败:', err);
    this.isLoading = false;
    
    promptAction.showToast({
      message: err.message || '登录失败，请重试',
      duration: 2000
    });
  });
}
```

## 执行流程

```
1. 用户点击登录按钮
   ↓
2. handleLogin() 调用 AuthService.login()
   ↓
3. Promise 回调中：
   - 保存 loginUserData
   - 设置 loginSuccessful = true
   ↓
4. @Watch 装饰器检测到 loginSuccessful 状态变化
   ↓
5. ArkUI 框架在 UI 线程中调用 onLoginSuccess()
   ↓
6. onLoginSuccess() 中：
   - 显示 Toast 提示（成功）
   - 执行路由跳转（成功）
   ↓
7. 跳转到角色选择页
```

## 为什么这个方案有效？

### 1. UI 上下文保证
`@Watch` 装饰器的回调由 ArkUI 框架调用，**保证在 UI 上下文中执行**。

### 2. 响应式编程
符合 ArkTS 的响应式编程模式，状态变化自动触发 UI 更新。

### 3. 可靠性高
不依赖 setTimeout 等不可靠的 hack 方法。

## 失败的尝试

### 1. 直接在 Promise 回调中跳转
```typescript
// ❌ 失败：UI execution context not found
AuthService.login(loginRequest).then((response) => {
  router.pushUrl({...}); // 失败
});
```

### 2. 使用 setTimeout 延迟
```typescript
// ❌ 失败：仍然找不到 UI 上下文
AuthService.login(loginRequest).then((response) => {
  setTimeout(() => {
    router.pushUrl({...}); // 仍然失败
  }, 100);
});
```

### 3. 清除编译缓存
```typescript
// ❌ 无效：问题不在编译缓存
// 清除 .hvigor/cache 和 entry/.preview 后仍然失败
```

## 适用场景

这个模式适用于所有需要在异步操作后执行 UI 操作的场景：

### 1. 登录/注册后跳转
```typescript
@State @Watch('onAuthSuccess') authDone: boolean = false;
```

### 2. 数据加载后显示弹窗
```typescript
@State @Watch('onDataLoaded') dataReady: boolean = false;
```

### 3. 操作成功后刷新列表
```typescript
@State @Watch('onOperationSuccess') operationDone: boolean = false;
```

### 4. 错误处理
```typescript
@State @Watch('onError') hasError: boolean = false;
@State errorMessage: string = '';
```

## 注意事项

### 1. 避免无限循环
```typescript
// ❌ 错误：在 @Watch 回调中修改被监听的状态
onLoginSuccess() {
  this.loginSuccessful = false; // 会再次触发回调
  this.loginSuccessful = true;  // 无限循环！
}
```

### 2. 检查状态有效性
```typescript
// ✅ 正确：始终检查状态是否有效
onLoginSuccess() {
  if (this.loginSuccessful && this.loginUserData) {
    // 执行操作
  }
}
```

### 3. 处理异常
```typescript
// ✅ 正确：捕获并处理异常
onLoginSuccess() {
  if (this.loginSuccessful && this.loginUserData) {
    router.pushUrl({...})
      .catch((err: Error) => {
        console.error('跳转失败:', err);
        promptAction.showToast({
          message: '页面跳转失败',
          duration: 2000
        });
      });
  }
}
```
