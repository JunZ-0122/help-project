## 📊 核心数据模型

### User (用户)
```typescript
{
  id: string;
  name: string;
  phone: string;
  avatar?: string;
  role: 'help-seeker' | 'volunteer' | 'community';
  status: 'online' | 'offline' | 'busy';
}
```

### HelpRequest (求助请求)
```typescript
{
  id: string;
  type: 'medical' | 'shopping' | 'repair' | 'companion' | 'emergency';
  title: string;
  description: string;
  location: string;
  urgency: 'low' | 'medium' | 'high' | 'emergency';
  status: 'pending' | 'assigned' | 'in-progress' | 'completed' | 'cancelled';
  scheduledTime?: string;  // ISO 8601
  images?: string[];
}
```

### ChatMessage (聊天消息)
```typescript
{
  id: number;
  requestId: string;
  senderId: string;
  text: string;
  type: 'text' | 'voice' | 'image' | 'location';
  createdAt: string;  // ISO 8601
}
```

### VolunteerOrder (志愿者订单)
```typescript
{
  id: string;
  requestId: string;
  volunteerId: string;
  status: 'accepted' | 'in-progress' | 'completed' | 'cancelled';
  acceptedAt: string;
  completedAt?: string;
}
```

## 🔌 核心 API 接口

### 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": "2024-01-01T00:00:00Z"
}
```

### P0 接口（第一周）

#### 认证
- `POST /api/auth/login` - 登录
- `POST /api/auth/register` - 注册
- `POST /api/auth/send-code` - 发送验证码
- `POST /api/auth/refresh-token` - 刷新 Token

#### 求助请求
- `GET /api/requests` - 获取列表（支持分页和筛选）
- `POST /api/requests` - 创建求助
- `GET /api/requests/:id` - 获取详情
- `PUT /api/requests/:id` - 更新求助
- `DELETE /api/requests/:id` - 取消求助

#### 志愿者订单
- `GET /api/orders/my` - 获取我的订单
- `POST /api/orders/accept` - 接受订单
- `PUT /api/orders/:id/status` - 更新状态
- `POST /api/orders/:id/complete` - 完成订单
- `GET /api/orders/:id/progress` - 获取进度

### P1 接口（第二周）

#### 聊天消息
- `GET /api/messages/:requestId` - 获取历史
- `POST /api/messages` - 发送消息
- `PUT /api/messages/read` - 标记已读
- `WebSocket /ws` - 实时通信

#### 文件上传
- `POST /api/upload/image` - 上传图片
- `POST /api/upload/audio` - 上传语音

#### 社区管理
- `GET /api/community/requests/pending` - 待分配请求
- `GET /api/community/volunteers/available` - 可用志愿者
- `POST /api/community/assign` - 分配志愿者
- `GET /api/community/statistics` - 统计数据

## 🔄 业务流程

### 求助流程
```
求助者发布 → 社区审核/分配 → 志愿者接单 → 
提供服务 → 更新进度 → 完成 → 评价
```

### 状态流转
```
pending → assigned → in-progress → completed/cancelled
```

## 🔐 认证机制

- 使用 JWT Token
- Token 有效期: 2 小时
- RefreshToken 有效期: 7 天
- 请求头: `Authorization: Bearer {token}`

## 💾 数据库表

### 核心表
- `users` - 用户表
- `help_requests` - 求助请求表
- `volunteer_orders` - 志愿者订单表
- `chat_messages` - 聊天消息表
- `reviews` - 评价表

### 索引建议
- users: phone (唯一)
- help_requests: user_id, status, created_at
- volunteer_orders: volunteer_id, status
- chat_messages: request_id, created_at

## ⚡ 性能要求

- API 响应时间 < 500ms
- 支持 1000+ 并发用户
- 使用 Redis 缓存热点数据
- 图片使用 OSS/CDN

---

## 🛡️ 安全要求

- 密码使用 bcrypt 加密
- 防止 SQL 注入
- 防止 XSS 攻击
- API 限流
- HTTPS 加密

## 🔍 常见问题

**Q: 时间格式？**  
A: 统一使用 ISO 8601: "2024-01-01T10:00:00Z"

**Q: 分页参数？**  
A: page (从1开始), pageSize (默认10)

**Q: 图片 URL？**  
A: 返回完整 URL: "https://example.com/uploads/image.jpg"

**Q: 枚举值？**  
A: 严格按照前端模型定义，如 role: 'help-seeker' | 'volunteer' | 'community'
