## 经验总结

### ArkTS 开发经验
1. **严格模式要求高**：所有类型必须明确声明
2. **UI 上下文重要**：异步操作后的 UI 更新需要注意上下文
3. **路由跳转技巧**：使用 setTimeout 确保在正确的上下文中执行
4. **编译器缓存**：修改代码后记得清理缓存

### Spring Boot 开发经验
1. **三层架构清晰**：Controller → Service → Mapper
2. **统一响应格式**：使用 Result<T> 包装所有响应
3. **JWT 认证简单**：使用拦截器统一处理
4. **MyBatis 灵活**：XML 映射文件便于维护

### 前后端协作经验
1. **接口文档重要**：提前定义好接口格式
2. **数据模型一致**：前后端数据结构保持一致
3. **错误处理统一**：使用统一的错误码和错误信息
4. **测试数据完整**：准备好测试账号和数据
