# 无障碍设置手动测试指南

## 概述

本指南提供详细的步骤说明，帮助测试人员完成无障碍设置的手动验证工作。

**相关文档**: 
- 验证清单: `ACCESSIBILITY_VERIFICATION_CHECKLIST.md`
- 需求文档: `.kiro/specs/accessibility-global-state/requirements.md`
- 设计文档: `.kiro/specs/accessibility-global-state/design.md`

---

## 测试前准备

### 1. 环境准备
- [ ] 确保应用已正确编译和安装
- [ ] 准备测试设备（HarmonyOS 3.0+）
- [ ] 准备截图工具
- [ ] 打开验证清单文档，准备记录结果

### 2. 了解无障碍设置位置
- 求助者个人中心: `HelpSeekerHomePage` → 右上角个人图标 → `ProfilePage` → "无障碍设置"
- 志愿者个人中心: `VolunteerHomePage` → 右上角个人图标 → `ProfilePage` → "无障碍设置"

### 3. 了解预期效果

#### 大字体模式
- 所有文字大小增加 25%
- 例如：14px → 17.5px, 16px → 20px, 18px → 22.5px
- 布局应该自适应，不出现文字溢出或重叠

#### 高对比度模式
- 背景色: 黄色 (#ffff00)
- 文字颜色: 黑色 (#000000)
- 边框: 3px 黑色粗边框
- 字体: 加粗 (FontWeight.Bold)
- 移除阴影效果

## 测试流程

### 阶段 1：基础功能测试

#### 测试 1.1：启用大字体模式
1. 启动应用并登录
2. 进入个人中心（ProfilePage）
3. 点击"无障碍设置"展开设置面板
4. 找到"大字体模式"开关
5. **截图 1**: 开关关闭状态
6. 点击开关，启用大字体模式
7. **截图 2**: 开关开启状态
8. **观察**: 页面文字是否立即变大
9. **验证**: 
   - 开关状态是否正确显示为"开启"
   - 页面文字是否增大约 25%
   - 布局是否正常，无文字溢出

**预期结果**: 
- 开关立即切换到开启状态
- 页面所有文字立即增大
- 布局保持正常

#### 测试 1.2：启用高对比度模式
1. 在个人中心页面
2. 找到"高对比度模式"开关
3. **截图 3**: 开关关闭状态（正常配色）
4. 点击开关，启用高对比度模式
5. **截图 4**: 开关开启状态（高对比度配色）
6. **观察**: 页面配色是否立即改变
7. **验证**:
   - 背景色是否变为黄色
   - 文字颜色是否变为黑色
   - 边框是否变为 3px 黑色
   - 字体是否加粗

**预期结果**:
- 开关立即切换到开启状态
- 页面立即应用黑黄配色
- 所有元素有明显的黑色粗边框
- 文字加粗清晰可读

#### 测试 1.3：同时启用两种模式
1. 在个人中心页面
2. 同时启用"大字体模式"和"高对比度模式"
3. **截图 5**: 两种模式同时启用
4. **验证**:
   - 文字既增大又加粗
   - 配色为黑黄高对比度
   - 边框为 3px 黑色
   - 布局正常

**预期结果**:
- 两种模式效果叠加
- 文字大且清晰
- 高对比度配色应用正确

### 阶段 2：页面切换一致性测试

#### 测试 2.1：求助者页面流程
1. 在求助者个人中心启用所有无障碍设置
2. 导航到以下页面，每个页面都要验证：
   - 返回首页（HelpSeekerHomePage）
   - 进入紧急求助（EmergencyRequestPage）
   - 返回首页，进入选择求助类型（RequestTypePage）
   - 返回首页，进入我的求助（MyRequestsPage）
   - 点击一个求助项，进入求助详情（RequestDetailPage）

3. **对每个页面执行以下验证**:
   - [ ] 页面加载后立即显示无障碍设置效果
   - [ ] 大字体模式生效（文字增大 25%）
   - [ ] 高对比度模式生效（黑黄配色、粗边框）
   - [ ] 布局正常，无错乱
   - [ ] 截图记录

4. **返回个人中心验证**:
   - [ ] 开关状态仍然为"开启"
   - [ ] 设置未被重置

**预期结果**:
- 所有页面立即应用无障碍设置
- 页面切换不影响设置状态
- 返回个人中心时开关状态保持

#### 测试 2.2：志愿者页面流程
1. 切换到志愿者角色
2. 在志愿者个人中心启用所有无障碍设置
3. 导航到以下页面，每个页面都要验证：
   - 返回首页（VolunteerHomePage）
   - 点击一个请求，进入请求详情（RequestDetailPage）
   - 返回首页，进入我的订单（MyOrdersPage）
   - 点击一个订单，进入订单详情（OrderDetailPage）
   - 返回首页，进入进度跟踪（ProgressPage）
   - 进入聊天页面（ChatPage）
   - 进入评价页面（ReviewPage）

4. **对每个页面执行相同的验证**（同测试 2.1）

5. **返回个人中心验证**（同测试 2.1）

**预期结果**: 同测试 2.1

#### 测试 2.3：社区管理页面流程
1. 切换到社区管理员角色
2. 在个人中心启用所有无障碍设置（如果有）
3. 导航到以下页面：
   - 社区首页（CommunityHomePage）
   - 分配页面（AssignPage）
   - 管理页面（ManagePage）
   - 请求详情（RequestDetailPage）
   - 统计页面（StatisticsPage）

4. **对每个页面执行相同的验证**

**预期结果**: 同测试 2.1

#### 测试 2.4：公共页面流程
1. 在任意角色的个人中心启用所有无障碍设置
2. 导航到以下页面：
   - 角色选择页面（RoleSelectPage）
   - 退出登录，查看登录页面（LoginPage）

#### 测试 3.1：应用重启后设置保持
1. 在个人中心启用所有无障碍设置
2. **截图 6**: 设置启用状态
3. 完全关闭应用（从后台清除）
4. 重新启动应用
5. 登录并进入个人中心
6. **截图 7**: 重启后的设置状态
7. **验证**:

   - [ ] 页面立即应用无障碍设置

- 设置从本地存储正确加载

3. 关闭应用
4. 重新启动应用
5. **验证**: 大字体模式仍然启用

7. 导航到 2-3 个页面验证禁用
8. 关闭应用
9. 重新启动应用
10. **验证**: 大字体模式仍然禁用

**预期结果**:
- 每次设置变更都被正确保存
- 应用重启后加载最新设置

## 常见问题排查

### 问题 1：设置不生效
**症状**: 点击开关后页面样式没有变化

**排查步骤**:
1. 检查开关状态是否正确切换
2. 检查控制台是否有错误日志
3. 检查页面是否使用了 @StorageLink 装饰器
4. 检查 AccessibilityManager 是否正确初始化

### 问题 2：页面切换后设置丢失
**症状**: 切换到新页面后无障碍设置失效

**排查步骤**:
1. 检查新页面是否添加了 @StorageLink 装饰器
2. 检查 AppStorage 中的值是否正确
3. 检查页面是否正确应用了条件样式

### 问题 3：应用重启后设置丢失
**症状**: 重启应用后设置恢复为默认值

**排查步骤**:
1. 检查 EntryAbility 是否调用了 AccessibilityManager.init()
2. 检查本地存储是否有数据
3. 检查 StorageUtil 的读写是否正常

### 问题 4：布局错乱
**症状**: 启用大字体模式后文字溢出或重叠

**排查步骤**:
1. 检查容器是否有固定高度
2. 检查是否使用了 layoutWeight 或 flexGrow
3. 检查文字是否设置了 maxLines 和 textOverflow
