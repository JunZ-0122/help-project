# 页面显示问题修复

## 问题描述

应用启动后页面没有显示。

## 问题原因

`main_pages.json` 路由配置文件中缺少 `pages/Index` 页面的注册，导致 `EntryAbility` 无法加载入口页面。

## 修复内容

### 1. 添加 Index 页面到路由配置

**文件**: `help_system/entry/src/main/resources/base/profile/main_pages.json`

**修复前**:
```json
{
  "src": [
    "pages/SplashPage",
    "pages/LoginPage",
    ...
  ]
}
```

**修复后**:
```json
{
  "src": [
    "pages/Index",  // ← 添加入口页面
    "pages/SplashPage",
    "pages/LoginPage",
    ...
  ]
}
```

### 2. 优化状态栏配置代码

**文件**: `help_system/entry/src/main/ets/entryability/EntryAbility.ets`

简化了状态栏配置的错误处理逻辑，使用 Promise 链式调用替代回调函数。

**优化后的代码**:
```typescript
windowStage.getMainWindow().then((windowClass) => {
  // 显示状态栏
  windowClass.setWindowSystemBarEnable(['status']).then(() => {
    // 设置状态栏样式
    const systemBarProperties: window.SystemBarProperties = {
      statusBarColor: '#ffffff',
      statusBarContentColor: '#000000',
      isStatusBarLightIcon: false
    };
    
    windowClass.setWindowSystemBarProperties(systemBarProperties);
  });
}).catch((err: Error) => {
  hilog.error(DOMAIN, 'testTag', 'Failed to configure status bar. Cause: %{public}s', JSON.stringify(err));
});
```

## 页面路由配置说明

### main_pages.json 的作用

`main_pages.json` 文件用于注册应用中所有可用的页面路由。只有在此文件中注册的页面才能被 `router` 导航。

### 页面注册规则

1. **必须注册**: 所有需要通过 `router.pushUrl()` 或 `router.replaceUrl()` 访问的页面
2. **路径格式**: 相对于 `ets/pages/` 目录的路径，不需要 `.ets` 后缀
3. **顺序**: 第一个页面通常是应用的入口页面

## Index 页面的作用

`pages/Index.ets` 是应用的入口页面，主要功能：

1. **作为启动入口**: 被 `EntryAbility` 首先加载
2. **配置初始化**: 可以在这里进行全局配置
3. **路由跳转**: 自动跳转到实际的首页（SplashPage）

```typescript
@Entry
@Component
struct Index {
  aboutToAppear() {
    // 应用启动时直接跳转到启动页
    router.replaceUrl({
      url: 'pages/SplashPage'
    });
  }

  build() {
    Column() {
      Text('加载中...')
        .fontSize(18)
        .fontColor('#666666')
    }
    .width('100%')
    .height('100%')
    .backgroundColor('#f9fafb')
    .justifyContent(FlexAlign.Center)
  }
}
```

## 常见问题

### Q1: 为什么需要 Index 页面？

**A**: Index 页面作为应用的统一入口，有以下优势：
- 便于进行全局初始化
- 可以根据条件跳转到不同页面
- 符合 HarmonyOS 应用开发规范

### Q2: 可以直接加载 SplashPage 吗？

**A**: 可以，但不推荐。修改 `EntryAbility.ets`:
```typescript
windowStage.loadContent('pages/SplashPage', (err) => {
  // ...
});
```

但这样会失去 Index 页面的灵活性。

### Q3: 添加新页面后需要做什么？

**A**: 必须在 `main_pages.json` 中注册新页面：
```json
{
  "src": [
    "pages/Index",
    "pages/NewPage",  // ← 添加新页面
    ...
  ]
}
```

### Q4: 页面路径写错会怎样？

**A**: 应用会报错并无法启动。常见错误：
- ❌ `pages/Index.ets` - 不要加 `.ets` 后缀
- ❌ `/pages/Index` - 不要加前导斜杠
- ✅ `pages/Index` - 正确格式
