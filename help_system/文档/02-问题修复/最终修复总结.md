### 2. UI 执行上下文丢失错误 (5 处)

**问题描述**:
```
Error: Internal error. UI execution context not found.
```

**原因**:
在页面跳转后尝试在原页面显示 Toast，此时 UI 上下文已销毁

**修复原则**:
- ❌ 不要在页面跳转后显示 Toast
- ✅ 页面跳转本身就是最好的反馈
- ✅ 只在跳转失败时显示错误提示

**修复的文件** (6个):
1. ✅ `LoginPage.ets` - 登录成功后跳转
2. ✅ `EmergencyRequestPage.ets` - 紧急求助提交后跳转
3. ✅ `help-seeker/RequestTypePage.ets` - 求助发布后跳转
4. ✅ `community/AssignPage.ets` - 分配成功后返回
5. ✅ `community/RequestDetailPage.ets` - 取消求助后返回
6. ✅ `volunteer/RequestDetailPage.ets` - 接单成功后跳转

**修复示例**:
```typescript
// ❌ 错误做法
setTimeout(() => {
  this.isSubmitting = false;
  promptAction.showToast({
    message: '提交成功！',
    duration: 2000
  });
  
  setTimeout(() => {
    router.pushUrl({ url: 'NextPage' });
  }, 500);
}, 1000);

// ✅ 正确做法
setTimeout(() => {
  this.isSubmitting = false;
  
  router.pushUrl({ url: 'NextPage' })
    .catch((err: Error) => {
      console.error('导航失败:', err);
      promptAction.showToast({
        message: '跳转失败，请重试',
        duration: 2000
      });
    });
}, 1000);
```

## 🎯 关键修复要点

### 1. ArkTS 严格模式规则
- 所有对象字面量必须有明确的类型
- 嵌套对象也需要定义接口
- 数组必须有明确的元素类型
- 状态变量必须指定类型

### 2. UI 上下文管理
- 页面跳转后不要执行 UI 操作
- Toast 提示要在跳转前显示
- 或者只在跳转失败时显示错误

### 3. HarmonyOS 特性
- 使用 `Scroll` 而不是 `ScrollView`
- 使用 `.linearGradient()` 方法
- 路由传参使用 `params` 对象
- 颜色使用十六进制数字格式
