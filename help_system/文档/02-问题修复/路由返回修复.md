# 🔧 router.back() UI 上下文错误修复

## 问题描述

**错误信息**: `Error: Parameter error. UI execution context not found.`  
**发生位置**: `community/AssignPage.ets` 的 `handleAssign()` 方法  
**错误原因**: 在 `setTimeout` 回调中调用 `router.back()` 时，UI 上下文已经不可用

## 问题分析

### 错误代码
```typescript
handleAssign() {
  this.isAssigning = true;

  setTimeout(() => {
    this.isAssigning = false;
    
    // ❌ 在 setTimeout 回调中调用 router.back()
    // 此时 UI 上下文可能已经不可用
    router.back();
  }, 1000);
}
```

### 为什么会失败？

1. **UI 上下文生命周期**: 当页面状态改变或即将销毁时，UI 上下文可能变得不可用
2. **异步操作的风险**: `setTimeout` 是异步的，回调执行时页面状态可能已经改变
3. **路由方法的要求**: `router.back()` 等路由方法需要有效的 UI 上下文才能执行

## 修复方案

### 方案 1: 立即调用路由方法（推荐）

```typescript
handleAssign() {
  if (!this.selectedVolunteer) {
    promptAction.showToast({
      message: '请选择志愿者',
      duration: 2000
    });
    return;
  }

  // 显示分配中状态
  this.isAssigning = true;

  // 短暂延迟后重置状态（不影响路由）
  setTimeout(() => {
    this.isAssigning = false;
  }, 100);

  // ✅ 立即返回上一页，不在 setTimeout 中调用
  router.back();
}
```

### 方案 2: 使用 Promise（如果需要等待）

```typescript
async handleAssign() {
  if (!this.selectedVolunteer) {
    promptAction.showToast({
      message: '请选择志愿者',
      duration: 2000
    });
    return;
  }

  this.isAssigning = true;

  // 模拟异步操作
  await new Promise(resolve => setTimeout(resolve, 800));
  
  this.isAssigning = false;

  // ✅ 在 async 函数中直接调用
  router.back();
}
```

## 核心原则

### ❌ 不要这样做
```typescript
// 1. 在 setTimeout 回调中调用路由方法
setTimeout(() => {
  router.back();
}, 1000);

// 2. 在 setTimeout 回调中调用 router.pushUrl
setTimeout(() => {
  router.pushUrl({ url: 'NextPage' });
}, 1000);
```

### ✅ 应该这样做
```typescript
// 1. 立即调用路由方法
router.back();

// 2. 或者在 async 函数中使用 await
async function navigate() {
  await someAsyncOperation();
  router.back();
}

// 3. 使用 catch 处理错误
router.pushUrl({ url: 'NextPage' })
  .catch(err => {
    console.error('导航失败:', err);
  });
```

1. **路由操作应该同步执行**: 不要在异步回调中调用路由方法

4. **用户反馈**: 页面切换本身就是最好的反馈，不需要额外的 Toast
