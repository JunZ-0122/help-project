### 问题描述
HarmonyOS 的路由系统不支持在 URL 中使用查询参数（如 `?id=1`），必须使用 `params` 对象传递参数。

### 修复前后对比

#### ❌ 错误写法
```typescript
// 方式1：直接在 URL 中拼接参数
router.pushUrl({
  url: `pages/RequestDetailPage?id=${id}`
});

// 方式2：通过 navigateToPage 方法
this.navigateToPage(`pages/RequestDetailPage?id=${id}`);
```

#### ✅ 正确写法
```typescript
router.pushUrl({
  url: 'pages/RequestDetailPage',
  params: {
    id: id,
    // 可以传递多个参数
    title: title,
    status: status
  }
}).catch((err: Error) => {
  console.error('导航失败:', err);
});
```

### 在目标页面接收参数

```typescript
@Entry
@Component
struct RequestDetailPage {
  @State requestId: string = '';
  @State requestTitle: string = '';
  
  aboutToAppear() {
    // 获取路由参数
    const params = router.getParams() as Record<string, string>;
    this.requestId = params['id'] || '';
    this.requestTitle = params['title'] || '';
    
    console.info('接收到的参数:', {
      id: this.requestId,
      title: this.requestTitle
    });
  }
  
  build() {
    Column() {
      Text(`请求ID: ${this.requestId}`)
      Text(`标题: ${this.requestTitle}`)
    }
  }
}
```

### 关键要点

1. **不要在 URL 中使用查询参数** - HarmonyOS 不支持 `?key=value` 格式
2. **使用 params 对象** - 通过 `params` 传递所有参数
3. **类型安全** - 在接收端使用类型断言确保类型安全
4. **错误处理** - 始终添加 `.catch()` 处理路由失败
5. **日志记录** - 使用 `console.info/error` 记录关键信息

### 注意事项

- 参数会被序列化，复杂对象需要先转换为 JSON 字符串
- 参数大小有限制，不要传递过大的数据
- 敏感信息不应通过路由参数传递
