# 后端开发需求文档

## 项目概述

**项目名称**：弱势群体互助系统 (Help System)

**项目描述**：一个连接弱势群体、志愿者和社区管理员的互助平台，支持求助发布、志愿者接单、实时聊天、进度跟踪等功能。

**前端技术栈**：HarmonyOS (ArkTS)

**后端要求**：RESTful API + WebSocket

---

## 系统角色

### 1. 弱势群体 (help-seeker)
- 发布求助请求
- 查看求助状态
- 与志愿者聊天
- 评价志愿者服务

### 2. 志愿者 (volunteer)
- 浏览求助列表
- 接受求助订单
- 更新服务进度
- 与求助者聊天

### 3. 社区管理员 (community)
- 管理求助请求
- 分配志愿者
- 查看统计数据
- 监控系统运行

## API 接口清单

### 基础配置

**API 基础地址**：`https://your-api-server.com/api`

**认证方式**：Bearer Token (JWT)

**请求头**：
```
Content-Type: application/json
Authorization: Bearer {token}
```

**统一响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": "2024-01-01T00:00:00Z"
}
```

**分页响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "totalPages": 10
  },
  "timestamp": "2024-01-01T00:00:00Z"
}
```

## 1. 认证模块 (Auth)

### 1.1 用户登录
- **接口**：`POST /api/auth/login`
- **请求体**：
```json
{
  "phone": "13800138000",
  "password": "password123",
  "loginType": "password"
}
```
- **响应**：
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "refresh_token_string",
  "expiresIn": 7200,
  "user": {
    "id": "user_001",
    "name": "张三",
    "phone": "13800138000",
    "avatar": "https://example.com/avatar.jpg",
    "role": "help-seeker",
    "status": "active"
  }
}
```

### 1.2 用户注册
- **接口**：`POST /api/auth/register`
- **请求体**：
```json
{
  "phone": "13800138000",
  "password": "password123",
  "verificationCode": "123456",
  "name": "张三",
  "role": "help-seeker"
}
```

### 1.3 发送验证码
- **接口**：`POST /api/auth/send-code`
- **请求体**：
```json
{
  "phone": "13800138000",
  "type": "login"
}
```

### 1.4 刷新 Token
- **接口**：`POST /api/auth/refresh-token`
- **请求体**：
```json
{
  "refreshToken": "refresh_token_string"
}
```

### 1.5 退出登录
- **接口**：`POST /api/auth/logout`

## 2. 求助请求模块 (Request)

### 2.1 获取求助列表
- **接口**：`GET /api/requests`
- **查询参数**：
  - `page`: 页码 (默认 1)
  - `pageSize`: 每页数量 (默认 10)
  - `type`: 求助类型 (可选)
  - `status`: 状态 (可选)
  - `urgency`: 紧急程度 (可选)
- **响应**：分页格式，items 为 HelpRequest 数组

### 2.2 创建求助
- **接口**：`POST /api/requests`
- **请求体**：
```json
{
  "type": "medical",
  "title": "需要陪同就医",
  "description": "明天上午需要去医院检查",
  "location": "北京市朝阳区某某医院",
  "latitude": 39.9042,
  "longitude": 116.4074,
  "urgency": "high",
  "scheduledTime": "2024-01-02T09:00:00Z",
  "images": ["https://example.com/image1.jpg"]
}
```

### 2.3 获取求助详情
- **接口**：`GET /api/requests/:id`
- **响应**：HelpRequest 对象

### 2.4 更新求助
- **接口**：`PUT /api/requests/:id`
- **请求体**：部分 HelpRequest 字段

### 2.5 取消求助
- **接口**：`DELETE /api/requests/:id`

### 2.6 获取我的求助列表
- **接口**：`GET /api/requests/my`
- **查询参数**：同 2.1

## 4. 聊天消息模块 (Message)

### 4.1 获取聊天历史
- **接口**：`GET /api/messages/:requestId`
- **查询参数**：
  - `page`: 页码
  - `pageSize`: 每页数量
- **响应**：ChatMessage 数组

### 4.2 发送消息
- **接口**：`POST /api/messages`
- **请求体**：
```json
{
  "requestId": "req_001",
  "receiverId": "user_002",
  "text": "你好，我已经在路上了",
  "type": "text"
}
```

### 4.3 标记消息已读
- **接口**：`PUT /api/messages/read`
- **请求体**：
```json
{
  "messageIds": [1, 2, 3]
}
```

### 4.4 WebSocket 实时消息
- **连接地址**：`ws://your-server-url/ws`
- **连接参数**：
  - Header: `Authorization: Bearer {token}`
- **消息格式**：
```json
{
  "type": "message",
  "data": {
    "id": 1,
    "requestId": "req_001",
    "senderId": "user_001",
    "senderName": "张三",
    "text": "你好",
    "time": "14:30",
    "type": "text",
    "createdAt": "2024-01-01T14:30:00Z"
  }
}
```

## 错误码定义

```
200: 成功
400: 请求参数错误
401: 未授权（token 无效或过期）
403: 禁止访问（权限不足）
404: 资源不存在
409: 资源冲突（如重复创建）
500: 服务器内部错误
```

**错误响应格式**：
```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null,
  "timestamp": "2024-01-01T00:00:00Z"
}
```

## 数据库设计建议

### 核心表

1. **users** - 用户表
2. **help_requests** - 求助请求表
3. **volunteer_orders** - 志愿者订单表
4. **chat_messages** - 聊天消息表
5. **reviews** - 评价表
6. **notifications** - 通知表
7. **emergency_contacts** - 紧急联系人表
8. **user_settings** - 用户设置表

### 关系说明
- 一个求助请求可以有一个志愿者订单
- 一个订单可以有多条聊天消息
- 一个订单可以有一条评价
- 一个用户可以有多个紧急联系人

## 安全要求

1. **密码加密**：使用 bcrypt 或类似算法
2. **Token 管理**：JWT，有效期 2 小时，支持刷新
3. **HTTPS**：所有 API 必须使用 HTTPS
4. **输入验证**：所有用户输入必须验证和过滤
5. **SQL 注入防护**：使用参数化查询
6. **XSS 防护**：对输出内容进行转义
7. **CORS 配置**：正确配置跨域请求
8. **限流**：API 请求限流，防止滥用

---

## 性能要求

1. **响应时间**：API 响应时间 < 500ms
2. **并发支持**：支持至少 1000 并发用户
3. **数据库优化**：合理使用索引
4. **缓存策略**：使用 Redis 缓存热点数据
5. **文件存储**：使用 OSS 或 CDN 存储图片和语音

---

## 测试要求

1. **单元测试**：核心业务逻辑覆盖率 > 80%
2. **接口测试**：所有 API 接口都需要测试
3. **压力测试**：模拟高并发场景
4. **安全测试**：SQL 注入、XSS 等安全测试

---

## 部署要求

1. **环境**：开发环境、测试环境、生产环境
2. **数据库**：MySQL 5.7+ 或 PostgreSQL 12+
3. **缓存**：Redis 6.0+
4. **消息队列**：RabbitMQ 或 Kafka（可选）
5. **日志**：集中式日志管理
