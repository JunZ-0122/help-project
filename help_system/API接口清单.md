## 1. 认证模块 (Auth)

| 接口 | 方法 | 路径 | 优先级 | 说明 |
|------|------|------|--------|------|
| 用户登录 | POST | /api/auth/login | P0 | 手机号+密码登录 |
| 用户注册 | POST | /api/auth/register | P0 | 新用户注册 |
| 发送验证码 | POST | /api/auth/send-code | P0 | 发送短信验证码 |
| 刷新Token | POST | /api/auth/refresh-token | P0 | 刷新访问令牌 |
| 退出登录 | POST | /api/auth/logout | P1 | 退出登录 |

## 接口依赖关系

```
认证模块 (基础)
    ↓
求助请求模块 + 志愿者订单模块 (核心业务)
    ↓
聊天消息模块 + 社区管理模块 (扩展功能)
    ↓
评价模块 + 用户模块 (辅助功能)
```

---

## 注意事项

1. 所有接口需要 Token 认证（除登录、注册、发送验证码）
2. 时间格式统一使用 ISO 8601
3. 分页参数: page (从1开始), pageSize (默认10)
4. 图片 URL 返回完整地址
5. 枚举值严格按照前端模型定义
