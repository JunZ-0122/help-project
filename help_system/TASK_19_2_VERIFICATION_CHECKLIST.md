# 任务 19.2 验证检查清单

## 任务信息
- **任务编号**: 19.2
- **任务名称**: 验证持久化功能
- **验证需求**: 4.2.1, 4.2.2

## 快速验证步骤

### ✅ 步骤 1：启动应用并设置无障碍选项
1. 在设备或模拟器上启动应用
2. 登录（求助者或志愿者角色）
3. 导航到个人中心页面
4. 找到"无障碍设置"面板
5. 启用以下设置：
   - ✅ 大字体模式
   - ✅ 高对比度模式
6. 验证页面立即应用了这些设置

### ✅ 步骤 2：关闭应用
1. 完全关闭应用（不是切换到后台）
2. 从最近任务列表中移除应用
3. 确保应用进程已终止

### ✅ 步骤 3：重新启动应用
1. 重新打开应用
2. 登录相同的账号
3. 导航到个人中心页面

### ✅ 步骤 4：验证设置已恢复
验证以下内容：
- [ ] 大字体模式开关仍然是**打开**状态
- [ ] 高对比度模式开关仍然是**打开**状态
- [ ] 页面文字显示为大字体（约增大 25%）
- [ ] 页面使用黑黄配色方案
- [ ] 文字为黑色加粗
- [ ] 边框为 3px 黑色粗边框

### ✅ 步骤 5：验证其他页面
1. 导航到首页
2. 导航到列表页（我的求助/我的订单）
3. 导航到详情页

验证：
- [ ] 所有页面都应用了大字体模式
- [ ] 所有页面都应用了高对比度模式
- [ ] 页面切换时设置保持一致

## 预期结果

### ✅ 成功标准
- 应用重启后，所有无障碍设置都正确恢复
- 开关状态与实际应用的样式一致
- 所有页面都正确应用了设置
- 页面切换时设置保持不变

### ❌ 失败情况
如果出现以下情况，说明持久化功能有问题：
- 重启后设置恢复为默认值
- 开关状态与实际样式不一致
- 某些页面没有应用设置
- 页面切换时设置丢失

在应用启动时，应该看到以下日志：
```

```

```
[INFO] 无障碍设置保存成功
```

## 故障排查

### 问题：设置没有保存
**检查**：
1. 查看日志是否有"保存无障碍设置失败"错误
2. 验证 StorageUtil.saveString 方法是否正常工作
3. 检查应用是否有存储权限

### 问题：设置保存了但没有恢复
**检查**：
1. 验证 EntryAbility.onCreate 中是否调用了 init()
2. 查看日志是否有"初始化无障碍设置失败"错误
3. 检查本地存储中的数据格式是否正确

### 问题：开关状态与样式不一致
**检查**：
1. 验证 @StorageLink 绑定的键名是否正确
2. 检查页面组件是否使用了 @StorageLink 装饰器
3. 验证 syncToAppStorage 方法是否正确执行
