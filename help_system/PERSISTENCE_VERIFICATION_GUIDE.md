# 无障碍设置持久化功能验证指南

## 验证目标

验证无障碍设置能够正确保存到本地存储，并在应用重启后恢复。

**验证需求：**
- 需求 4.2.1：设置能够保存到本地存储
- 需求 4.2.2：应用启动时能够加载保存的设置

## 前置条件

1. 应用已成功编译并安装到设备或模拟器
2. AccessibilityManager 已在 EntryAbility 中初始化
3. 求助者和志愿者的个人中心页面都有无障碍设置面板

## 验证步骤

### 测试场景 1：大字体模式持久化

1. **启动应用**
   - 打开应用并登录（求助者或志愿者角色）
   - 导航到个人中心页面

2. **启用大字体模式**
   - 找到"无障碍设置"面板
   - 打开"大字体模式"开关
   - ✅ 验证：页面上的文字立即变大（约增大 25%）

3. **关闭应用**
   - 完全关闭应用（不是切换到后台）
   - 从最近任务中移除应用

4. **重新启动应用**
   - 重新打开应用
   - 登录并导航到个人中心页面
   - ✅ 验证：大字体模式开关仍然是打开状态
   - ✅ 验证：页面上的文字仍然是大字体显示

5. **导航到其他页面**
   - 导航到首页、列表页等其他页面
   - ✅ 验证：所有页面的文字都是大字体显示

## 常见问题排查

### 问题 1：设置没有保存

**症状：** 修改设置后重启应用，设置恢复为默认值

**可能原因：**
1. StorageUtil 保存失败
2. AccessibilityManager 的 saveSettings 方法没有被调用
3. 本地存储权限问题

**排查步骤：**
1. 检查日志中是否有"保存无障碍设置失败"的错误信息
2. 在 updateLargeFont/updateHighContrast/updateVibration 方法中添加日志
3. 验证 StorageUtil.set 方法是否正常工作

### 问题 2：设置保存了但没有恢复

**症状：** 设置已保存到本地，但重启后没有加载

**可能原因：**
1. EntryAbility 中没有调用 AccessibilityManager.init()
2. init() 方法执行失败
3. loadSettings 方法解析 JSON 失败

**排查步骤：**
1. 检查 EntryAbility.onCreate 中是否调用了 init()
2. 检查日志中是否有"初始化无障碍设置失败"的错误信息
3. 验证本地存储中的 JSON 格式是否正确

### 问题 3：开关状态与实际样式不一致

**症状：** 开关显示已启用，但页面样式没有应用

**可能原因：**
1. @StorageLink 绑定的键名不正确
2. AppStorage 和 UI 状态不同步
3. 页面没有正确使用 @StorageLink

**排查步骤：**
1. 验证 STORAGE_KEYS 常量的键名
2. 检查页面组件是否使用了 @StorageLink 装饰器
3. 验证 syncToAppStorage 方法是否正确执行

## 技术细节

### 存储格式

设置以 JSON 格式存储在本地：

```json
{
  "largeFontEnabled": true,
  "highContrastEnabled": false,
  "vibrationEnabled": true
}
```

**存储键名：** `accessibility_settings`

### AppStorage 键名

- `accessibility_large_font` - 大字体模式
- `accessibility_high_contrast` - 高对比度模式
- `accessibility_vibration` - 震动反馈

### 初始化流程

```
应用启动
  ↓
EntryAbility.onCreate()
  ↓
AccessibilityManager.init()
  ↓
loadSettings() - 从本地存储读取
  ↓
syncToAppStorage() - 同步到 AppStorage
  ↓
页面组件通过 @StorageLink 获取初始值
```
