## 问题描述

```

```

## 深层原因分析

### 可能的原因

1. **HarmonyOS 版本问题**
   - 某些版本的 HarmonyOS SDK 在异步回调中的路由跳转有 bug
   - 建议检查 SDK 版本是否是最新的

2. **Preview 模式限制**
   - Preview 模式可能对路由跳转有限制
   - 建议在真机或模拟器上测试

3. **页面注册问题**
   - 检查 `main_pages.json` 中是否正确注册了 `RoleSelectPage`

4. **UI 线程问题**
   - Promise 回调可能不在主 UI 线程中执行
   - 需要使用特定的 API 切换到 UI 线程

## 推荐的长期解决方案

### 方案 1: 使用 UIAbility 的路由管理

在 `EntryAbility.ets` 中管理路由：

```typescript
// EntryAbility.ets
import { router } from '@kit.ArkUI';

export default class EntryAbility extends UIAbility {
  onWindowStageCreate(windowStage: window.WindowStage) {
    windowStage.loadContent('pages/LoginPage', (err, data) => {
      if (err.code) {
        return;
      }
      
      // 监听登录成功事件
      AppStorage.setOrCreate('loginSuccess', false);
      AppStorage.setOrCreate('loginUser', null);
    });
  }
}

// LoginPage.ets
AppStorage.setOrCreate('loginSuccess', true);
AppStorage.setOrCreate('loginUser', response.user);

// 在 EntryAbility 中监听并跳转
```

### 方案 2: 使用 EventHub

```typescript
// 在 EntryAbility 中
this.context.eventHub.on('loginSuccess', (user) => {
  router.pushUrl({
    url: 'pages/RoleSelectPage',
    params: { user }
  });
});

// 在 LoginPage 中
getContext(this).eventHub.emit('loginSuccess', response.user);
```

### 方案 3: 使用 @Watch 装饰器

```typescript
@State @Watch('onLoginSuccess') loginSuccessful: boolean = false;

onLoginSuccess() {
  if (this.loginSuccessful) {
    router.pushUrl({
      url: 'pages/RoleSelectPage'
    });
  }
}

// 在登录成功后
this.loginSuccessful = true;
```
