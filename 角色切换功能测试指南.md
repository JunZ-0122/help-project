# 角色切换自动跳转功能修复完成

更新时间：2026-03-14

## 问题描述

用户在个人中心切换身份后，点击角色卡片没有自动跳转到对应角色的主页，而是停留在角色选择页面。

## 问题根因

原代码使用 Promise 链式调用，错误处理不够完善，可能导致 Promise 链在某个环节失败后静默失败，没有执行后续的导航逻辑。

## 修复方案

### 代码改进

1. **从 Promise 链改为 async/await**
   - 更清晰的错误处理
   - 更容易调试和追踪执行流程

2. **改进存储 API 调用**
   - 从 `StorageUtil.getString('user')` + `JSON.parse()` 改为 `StorageUtil.get<UserProfile>('user')`
   - 减少手动解析 JSON 的错误风险

3. **增强日志记录**
   - 所有关键步骤都添加了 `[RoleSelect]` 前缀的日志
   - 便于追踪执行流程和定位问题

4. **改进导航逻辑**
   - 添加 500ms 延迟确保 Toast 显示
   - 添加导航成功/失败的回调处理
   - 明确的错误提示

### 修改的文件

- `help_system/entry/src/main/ets/pages/RoleSelectPage.ets`

### 关键代码变更

```typescript
// 修改前：Promise 链
handleRoleSelect(role: RoleInfo) {
  StorageUtil.getString('user')
    .then(userStr => {
      const currentUser = JSON.parse(userStr);
      // ... 复杂的 Promise 链
    });
}

// 修改后：async/await
async handleRoleSelect(role: RoleInfo) {
  try {
    console.log('[RoleSelect] Starting role selection:', role.id);
    
    const currentUser = await StorageUtil.get<UserProfile>('user');
    console.log('[RoleSelect] Current user:', currentUser);
    
    const hasRole = currentUser && !!currentUser.role;
    
    let updatedUser: UserProfile;
    if (hasRole) {
      updatedUser = await UserApi.switchUserRole(role.id);
    } else {
      updatedUser = await UserApi.updateUserRole(role.id);
    }
    
    await StorageUtil.set('user', updatedUser);
    this.showToast(`已选择：${role.title}`);
    
    setTimeout(() => {
      router.replaceUrl({ url: role.path })
        .then(() => console.log('[RoleSelect] Navigation successful'))
        .catch((err: Error) => {
          console.error('[RoleSelect] Navigation failed:', err);
          this.showToast('页面跳转失败');
        });
    }, 500);
    
  } catch (err) {
    console.error('[RoleSelect] Error:', err);
    this.showToast(error.message || '角色更新失败');
  }
}
```

### 常见问题排查

#### 问题 1：点击后没有任何反应

**可能原因**：
- 前端没有重新编译
- 预览器没有重启

**解决方法**：
1. 确保执行了 Clean → Rebuild
2. 完全停止预览器后重新启动

#### 问题 2：显示"角色更新失败"

**可能原因**：
- 后端服务未启动
- Token 已过期
- 网络连接问题

**解决方法**：
1. 检查后端是否在运行：`http://localhost:8080`
2. 查看后端日志是否有错误
3. 尝试重新登录

#### 问题 3：显示"页面跳转失败"

**可能原因**：
- 页面路径配置错误
- 页面未在 main_pages.json 中注册

**解决方法**：
1. 检查 `main_pages.json` 中是否包含目标页面
2. 查看控制台日志中的详细错误信息

#### 问题 4：看不到日志

**可能原因**：
- Log 窗口过滤设置不正确
- 日志级别设置过高

**解决方法**：
1. 在 Log 窗口中，确保选择了正确的设备/进程
2. 搜索 `[RoleSelect]` 关键字
3. 如果还是看不到，尝试使用 `console.error` 替代 `console.log`

## 技术细节

### API 调用流程

1. **首次选择角色**（用户没有 role）
   - 调用：`PUT /api/users/me/role`
   - 请求体：`{ "role": "volunteer" }`
   - 后端验证：用户必须没有角色才能调用此接口

2. **切换角色**（用户已有 role）
   - 调用：`PUT /api/users/me/role/switch`
   - 请求体：`{ "role": "help-seeker" }`
   - 后端验证：新角色不能与当前角色相同

### 页面路径映射

| 角色 ID | 角色名称 | 页面路径 |
|---------|---------|----------|
| help-seeker | 求助者 | pages/help-seeker/HelpSeekerHomePage |
| volunteer | 志愿者 | pages/volunteer/VolunteerHomePage |
| community | 社区管理 | pages/community/CommunityHomePage |
