## 3. 可用的 API 接口

### 3.1 认证接口
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/send-code` - 发送验证码
- `POST /api/auth/refresh-token` - 刷新Token
- `POST /api/auth/logout` - 退出登录

### 3.2 用户接口
- `GET /api/users/me` - 获取当前用户信息 ⭐
- `PUT /api/users/me` - 更新用户资料 ⭐
- `GET /api/users/{id}` - 获取用户信息
- `PUT /api/users/me/role` - 首次选择角色
- `PUT /api/users/me/role/switch` - 切换角色

### 3.3 求助请求接口
- `GET /api/requests` - 获取求助列表
- `POST /api/requests` - 创建求助
- `GET /api/requests/{id}` - 获取求助详情
- `GET /api/requests/my` - 获取我的求助列表
- `DELETE /api/requests/{id}` - 取消求助

### 3.4 志愿者订单接口
- `GET /api/volunteer/orders/my` - 获取我的订单列表 ⭐
- `POST /api/volunteer/orders/accept` - 接受求助请求
- `GET /api/volunteer/orders/{id}` - 获取订单详情
- `PUT /api/volunteer/orders/{id}/status` - 更新订单状态
- `POST /api/volunteer/orders/{id}/complete` - 完成订单

### 3.5 聊天消息接口
- `POST /api/chat/` - 发送消息
- `GET /api/chat/{requestId}` - 获取聊天历史
- `PUT /api/chat/read` - 标记消息为已读
- `GET /api/chat/{requestId}/unread` - 获取未读消息数量

### 3.6 评价接口
- `POST /api/reviews` - 创建评价
- `GET /api/reviews/order/{orderId}` - 根据订单ID获取评价

### 3.7 社区管理接口
- `GET /api/community/requests` - 获取所有求助请求
- `POST /api/community/assign` - 分配志愿者
- `GET /api/community/volunteers/available` - 获取可用志愿者列表

## 12. 故障排查

### 12.1 如果后端无法访问
1. 检查进程是否运行：`listProcesses`
2. 查看进程日志：`getProcessOutput(processId: 2)`
3. 检查端口占用：`netstat -ano | findstr 8080`

### 12.2 如果数据库连接失败
1. 检查 MySQL 服务是否启动
2. 检查数据库配置：`help/src/main/resources/application.yml`
3. 检查数据库是否存在：`mysql -u root -p -e "SHOW DATABASES;"`

### 12.3 如果前端无法连接后端
1. 检查后端是否启动：访问 `http://localhost:8080`
2. 检查前端 API 配置：`ApiService.ets` 中的 `API_BASE_URL`
3. 检查 CORS 配置：后端 `WebConfig.java`
