# 如何测试

本文说明如何本地启动并测试后端、前端及联调流程。

---

## 一、环境准备

### 1. 必需环境

| 环境 | 说明 |
|------|------|
| **MySQL** | 8.0，已创建数据库 `help`（见下方） |
| **JDK 17** | 后端运行 |
| **Node / Hvigor** | 前端构建（DevEco 自带） |
| **DevEco Studio** | 运行/预览 HarmonyOS 前端 |

### 2. 数据库

当前后端使用的库名是 **`help`**（见 `help/src/main/resources/application.yml`）。

若库尚未创建：

```sql
CREATE DATABASE IF NOT EXISTS help
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后执行项目中的建表脚本（若有 `help/src/main/resources/*.sql` 或项目文档中提到的 schema）。

## 四、前端配置与运行

### 1. API 地址

前端请求写死在 `help_system/entry/src/main/ets/services/ApiService.ets`：

```ts
const API_BASE_URL = 'http://172.31.32.1:8080/api';
```

- **模拟器 / 真机**：设备要能访问你电脑的 IP。把 `172.31.32.1` 改成你电脑在当前局域网的本机 IP（如 `ipconfig` 里的 IPv4），否则前端连不上后端。
- **本机浏览器/本地调试**：若只测本地，可改为 `http://localhost:8080/api`（模拟器/真机访问本机时一般仍建议用电脑 IP）。

### 2. 修改本机 IP 的方法

1. 在电脑上打开 CMD 或 PowerShell，执行 `ipconfig`。
2. 找到当前所用网络的 **IPv4 地址**（例如 `192.168.1.100`）。
3. 在 `ApiService.ets` 中把 `172.31.32.1` 改成该 IP，保存后重新编译运行前端。

### 3. 运行前端

1. 用 **DevEco Studio** 打开工程目录：`c:\Users\22789\Desktop\endProject\help_system`。
2. 连接模拟器或真机。
3. 点击运行（Run）或预览（Preview），安装并打开应用。

## 五、联调测试流程建议

### 1. 通用前置

1. 启动 **MySQL**，确认数据库 `help` 存在且有表。
2. 启动 **后端**（`help` 目录下 `.\mvnw.cmd spring-boot:run`）。
3. 确认前端 **API_BASE_URL** 指向正确的后端地址（本机 IP 或 localhost，见上）。
4. 在 DevEco 中运行/预览 **前端**。

### 2. 认证与角色

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 打开应用 → 启动页 → 登录页 | 进入登录页 |

| 7 | 管理列表某条待分配 → 「取消」 | 该条取消，列表刷新 |

### 6. 聊天

| 1 | 求助者/志愿者从订单或详情进入聊天 | 能看历史消息 |

## 六、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 前端一直加载或提示网络错误 | 前端连不上后端 | 检查后端是否已启动；把 ApiService 里地址改为本机 IP，且设备与电脑同网 |
| 登录提示 **Timeout was reached** | 请求发到错误 IP 或后端未启动 | **Previewer**：用 `127.0.0.1:8080`（已为默认）。**真机/模拟器**：改为本机局域网 IP（ipconfig 看 IPv4）。并确认后端已启动、防火墙放行 8080 |
| 登录提示 401 或失败 | 账号/密码错误或后端未启动 | 检查账号、后端日志；先用 PowerShell 调登录接口确认 |
| 模拟器/真机访问不到后端 | 使用了 localhost | 将 API_BASE_URL 改为电脑局域网 IP |
| 数据库连接失败 | 库名/账号/密码不对 | 核对 `help/src/main/resources/application.yml` 与 MySQL 实际配置（库名 `help`） |

### 1. 求助者详情页：空态与异常态

**1.1 无 ID 空态（无有效求助信息）**

- **操作**：不通过「我的求助」列表进入详情，而是直接打开详情页且不带求助 ID（若你项目里有从别处跳转详情且可能不传 id 的入口，从那里进；否则可暂时用路由/开发工具直接打开 `pages/help-seeker/RequestDetailPage` 且不传 params）。
- **预期**：页面显示「无有效求助信息」「请从列表页进入查看详情」和「返回列表」按钮，而不是空白或报错。

**1.2 加载失败 + 重试**

- **操作**：先关掉后端，或把 ApiService 里的地址改成错的，然后从「我的求助」点进某条求助详情。
- **预期**：先有 Toast 提示加载失败；页面上出现错误文案和「点击重试」按钮。
- **再操作**：恢复后端/正确地址后，点击「点击重试」。
- **预期**：重新请求，加载出详情内容。

### 2. 社区端：审核、取消、重分配

**2.1 审核通过 / 审核拒绝（详情页）**

- **操作**：用**社区管理员**账号登录 → 求助管理 → 找一条状态为「待分配」的求助 → 点「查看详情」。

- **操作**：点「审核通过」。
- **预期**：Toast「已通过审核」，详情刷新后状态变为「已通过审核」之类，底部操作区消失或变化。
- **再测**：另找一条「待分配」，点「审核拒绝」。
- **预期**：Toast「已拒绝」，状态变为「已拒绝」等。

**2.2 取消求助（详情页）**

- **操作**：在社区求助详情页（待分配或已分配均可），点「取消求助」。
- **预期**：Toast「已取消求助」，并自动返回上一页（或列表刷新）。

**2.3 重新分配志愿者（详情页）**

- **操作**：找一条**已分配**的求助，进详情。
- **预期**：底部有「重新分配志愿者」和「取消求助」。
- **操作**：点「重新分配志愿者」。
- **预期**：跳转到分配页，可重新选志愿者并分配。

**2.4 管理列表「取消」按钮**

- **操作**：社区 → 求助管理，找一条**待分配**的卡片。
- **预期**：卡片上有「查看详情」「分配志愿者」和**「取消」**按钮。
- **操作**：点该条卡片的「取消」。
- **预期**：Toast「已取消求助」，列表自动刷新，该条消失或状态变为已取消。

**2.5 管理列表空态与加载态**

- **操作**：在求助管理页，筛选到一种没有数据的条件（如选「已完成」且当前没有已完成），或清空数据后进入。
- **预期**：显示空态：图标 +「暂无求助记录」或「暂无符合条件的求助」，而不是空白。
- **操作**：从其他页切回求助管理、或刷新列表时。
- **预期**：加载过程中有转圈（LoadingProgress）和「加载中...」文案。

## 十、高德定位与志愿者匹配 测试

本节对应《高德定位与志愿者匹配-设计文档》实现后的联调与验证。

### 10.1 前置条件

- 数据库已执行扩展脚本：`help/src/main/resources/db/schema-extend.sql`（含 `user_locations` 等表）。
- 后端已配置高德 Web 服务 Key：`help/src/main/resources/application.yml` 中 `amap.web-service-key`（或环境变量 `AMAP_WEB_SERVICE_KEY`）。
- 前端已声明定位权限：`help_system/entry/src/main/module.json5` 中 `ohos.permission.APPROXIMATELY_LOCATION`、`ohos.permission.LOCATION`。

### 10.2 后端接口验证

| 接口 | 方法 | 说明 |
|------|------|------|
| 上报位置 | POST /api/users/me/location | Body: `{"latitude":39.9,"longitude":116.4,"address":"可选","source":"app"}`，需登录态 |
| 查询我的位置 | GET /api/users/me/location | 需登录态，无位置时返回错误 |
| 附近求助 | GET /api/volunteer/nearby-requests?page=1&pageSize=10 | 可选 query：latitude、longitude；不传则用当前用户最近上报位置，需登录态（志愿者） |
| 社区可用志愿者（按距离） | GET /api/community/volunteers/available?requestId=xxx 或 ?latitude=39.9&longitude=116.4 | 传 requestId 或 经纬度 时返回列表按距离排序 |

- **求助创建补全经纬度**：发布求助时仅传 `location` 不传 `latitude/longitude`，后端会调用高德地理编码补全并写入库，可查 `help_requests` 表确认 `latitude`、`longitude` 有值。

### 10.3 前端验证要点

| 场景 | 操作 | 预期 |
|------|------|------|
| 志愿者首页「附近求助」 | 登录为志愿者，进入首页；若已授权定位则先上报位置再拉取附近求助 | 列表为待接单且按距离排序，卡片展示「距离 X.X km」；若未授权或接口失败则降级为普通待接单列表（无距离） |
| 求助者发布求助 | 发布普通求助时授权定位 | 请求体带 `latitude`、`longitude`；未授权时仅地址，由后端 geocode 补全 |
| 社区分配页 | 进入某求助的分配志愿者页，接口带 requestId | 可用志愿者列表按与求助点距离排序，展示 distance |

### 10.4 快速检查清单（高德/匹配）

| 项目 | 操作摘要 | 通过 |
|------|----------|------|
| 位置上报 | 登录后调用 POST /api/users/me/location，再 GET 能查到 | ☐ |
| 附近求助 | 志愿者端首页拉取附近求助，有经纬度的求助显示距离 | ☐ |
| 发布带经纬度 | 求助者发布时授权定位，请求体含 lat/lng | ☐ |
| 仅地址发布 | 仅填地址不授权定位，后端能 geocode 补全经纬度 | ☐ |
| 社区按距离 | 分配页带 requestId，志愿者列表按距离排序 | ☐ |

### 10.5 获取真实位置（紧急求助「当前位置」）

- **真机（推荐）**  
  1. 用 USB 或无线连接 HarmonyOS 真机，在 DevEco 中选择该设备运行应用。  
  2. 进入紧急求助页，点「获取位置」，在弹框中允许定位权限。  
  3. 应用会拿到设备 GPS/网络定位，并自动填入逆地理得到的地址。

- **DevEco 模拟器**  
  1. 在模拟器窗口右侧找到 **扩展 / 更多**（⋯ 或 Location 图标）。  
  2. 进入 **「位置信息」/ Location**，选择「手动设置」输入经纬度，或从城市下拉框选城市。  
  3. 保存后回到应用，点「获取位置」；模拟器会把刚设置的坐标提供给应用，即可得到该位置对应的地址。

- **仅测表单时**  
  若不需要真实坐标，可点当前位置下方的灰色小字「模拟位置（测试）」填入北京坐标对应地址，或直接手动输入地址。
