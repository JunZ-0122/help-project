# 角色切换功能完整实现总结

## 需求背景

用户提出需求：**如果用户已经有角色，不想换账号，只想更换角色，该如何处理？**

原系统限制用户只能在首次选择角色，之后不允许修改。为了提升用户体验，实现了角色切换功能。

## 解决方案

采用**方案 A：添加角色切换接口**（推荐方案）

### 核心设计原则

1. **数据保留**：切换角色后保留所有历史数据
2. **权限变更**：切换后立即生效新角色权限
3. **二次确认**：切换前用户必须确认并了解影响
4. **审计追踪**：可扩展记录角色切换历史

### 与原接口的区别

| 特性 | 原接口 `/api/users/me/role` | 新接口 `/api/users/me/role/switch` |
|------|----------------------------|-----------------------------------|
| 用途 | 首次选择角色 | 切换已有角色 |
| 限制 | 已有角色不允许修改 | 允许切换到其他角色 |
| 验证 | 检查角色是否为空 | 检查是否切换到相同角色 |
| 使用场景 | 注册后选择角色 | 已有角色的用户切换 |

### 1. 后端实现

#### 新增接口

**文件**: `help/src/main/java/com/csi/help/controller/UserController.java`

```java
/**
 * 切换用户角色（允许已有角色的用户切换）
 */
@PutMapping("/me/role/switch")
public Result<User> switchRole(@RequestBody Map<String, String> params,
                                 @RequestAttribute("userId") String userId) {
    String newRole = params.get("role");
    
    // 验证角色值
    if (newRole == null || (!newRole.equals("help-seeker") && 
        !newRole.equals("volunteer") && !newRole.equals("community"))) {
        return Result.error("无效的角色类型");
    }
    
    // 检查用户是否存在
    User user = userService.getById(userId);
    if (user == null) {
        return Result.error("用户不存在");
    }
    
    // 检查是否切换到相同角色
    if (newRole.equals(user.getRole())) {
        return Result.error("当前已是该角色，无需切换");
    }
    
    // 更新角色
    userService.updateRole(userId, newRole);
    
    // 返回更新后的用户信息
    user = userService.getById(userId);
    user.setPassword(null);
    return Result.success(user);
}
```

**特点**:
- 复用现有的 `userService.updateRole()` 方法
- 复用现有的 `UserMapper.update()` SQL 语句
- 无需修改数据库表结构
- 无需修改 Service 层和 Mapper 层

## 智能角色选择逻辑（已完善）

### 功能说明

`RoleSelectPage` 现在支持智能判断用户状态，自动调用正确的 API 接口：

**逻辑流程**:
```
用户点击角色
    ↓
检查本地存储的 user.role
    ↓
┌─────────────────┬─────────────────┐
│  role 为空      │  role 不为空    │
│  (首次选择)     │  (切换角色)     │
├─────────────────┼─────────────────┤
│  调用           │  调用           │
│  updateUserRole │  switchUserRole │
│  接口           │  接口           │
└─────────────────┴─────────────────┘
    ↓                   ↓
更新本地存储 user 信息
    ↓
跳转到对应角色主页
```

**核心代码**（已修复类型错误）:
```typescript
async handleRoleSelect(roleId: string, path: string, title: string) {
  // 检查用户是否已有角色
  const userStr = await StorageUtil.getString('user');
  let hasRole = false;
  let currentRole = '';
  
  if (userStr) {
    try {
      // ✅ 修复：添加显式类型注解
      const user = JSON.parse(userStr) as UserProfile;
      currentRole = user.role || '';
      hasRole = currentRole !== '';
      console.info('[RoleSelectPage] 用户当前角色:', currentRole || '无');
    } catch (e) {
      console.error('[RoleSelectPage] 解析用户信息失败:', e);
    }
  }
  
  let updatedUser: UserProfile;
  
  if (hasRole) {
    // 用户已有角色，使用角色切换接口
    console.info('[RoleSelectPage] 用户已有角色，调用切换接口...');
    updatedUser = await UserApi.switchUserRole(roleId);
    console.info('[RoleSelectPage] 角色切换成功:', JSON.stringify(updatedUser));
  } else {
    // 用户无角色，使用首次选择接口
    console.info('[RoleSelectPage] 用户无角色，调用首次选择接口...');
    updatedUser = await UserApi.updateUserRole(roleId);
    console.info('[RoleSelectPage] 角色选择成功:', JSON.stringify(updatedUser));
  }
  
  // 更新本地存储的用户信息
  await StorageUtil.set('user', updatedUser);
  
  // 显示成功提示
  promptAction.showToast({
    message: `已选择：${title}`,
    duration: 1500
  });
  
  // 触发路由跳转
  this.targetPath = path;
  this.roleUpdateSuccess = true;
}
```

- ✅ 页面自动判断并调用正确的接口

## 问题修复记录 2：UI 上下文错误

### 问题：Toast 提示失败

**发现时间**: 2026-03-08 00:14

**问题描述**:
- **错误**: `Error: Internal error. UI execution context not found.`
- **错误代码**: `{"code":"100001"}`
- **位置**: `RoleSelectPage.handleRoleSelect()` 中的 `promptAction.showToast()`
- **影响**: 角色切换成功，但 Toast 提示无法显示

**成功的部分**:
- ✅ API 调用成功（200 响应）
- ✅ 数据库更新成功（role: community）
- ✅ 本地存储更新成功
- ✅ 后端逻辑完全正常

**失败的部分**:
- ❌ Toast 提示无法显示
- ❌ 页面跳转可能受影响

**根本原因**:
在 ArkTS 中，`async/await` 异步函数中调用 UI API（如 `promptAction.showToast()`）时，可能会失去 UI 执行上下文。

**错误代码**:
```typescript
async handleRoleSelect(roleId: string, path: string, title: string) {
  try {
    await UserApi.switchUserRole(roleId);  // 异步操作
    await StorageUtil.set('user', updatedUser);  // 异步操作
    
    // ❌ 在 async 函数中调用 UI API，失去上下文
    promptAction.showToast({
      message: `已选择：${title}`,
      duration: 1500
    });
  } catch (err) {
    // ...
  }
}
```

**修复方案**:
将 `async/await` 改为 Promise 链式调用，保持 UI 上下文连续性。

**修复后代码**:
```typescript
handleRoleSelect(roleId: string, path: string, title: string) {
  // 使用 Promise 链式调用，避免 async/await 导致的 UI 上下文丢失
  StorageUtil.getString('user')
    .then((userStr) => {
      // 判断是否已有角色
      let hasRole = false;
      if (userStr) {
        const user = JSON.parse(userStr) as UserProfile;
        hasRole = user.role !== '';
      }
      
      // 调用对应的 API
      if (hasRole) {
        return UserApi.switchUserRole(roleId);
      } else {
        return UserApi.updateUserRole(roleId);
      }
    })
    .then((updatedUser) => {
      // 更新本地存储
      return StorageUtil.set('user', updatedUser).then(() => {
        return Promise.resolve({ updatedUser, title, path });
      });
    })
    .then((result) => {
      // ✅ 在 Promise 链中显示 Toast（保持 UI 上下文）
      promptAction.showToast({
        message: `已选择：${result.title}`,
        duration: 1500
      });
      
      // 触发路由跳转
      this.targetPath = result.path;
      this.roleUpdateSuccess = true;
    })
    .catch((err) => {
      // 错误处理
      this.isUpdating = false;
      this.selectedRole = '';
      
      const error = err as Error;
      promptAction.showToast({
        message: error.message || '更新角色失败，请重试',
        duration: 2000
      });
    });
}
```
