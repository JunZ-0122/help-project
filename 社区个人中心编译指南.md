# 社区个人中心编译指南

**错误信息**: `Cannot find module '&entry/src/main/ets/pages/community/ProfilePage&'`

**原因**: DevEco Studio 预览器没有识别到新创建的 ProfilePage.ets 文件

## 解决方案

### 方法 1: 完整重新编译（推荐）

1. **停止预览器**
   - 点击 DevEco Studio 顶部的停止按钮（红色方块）

2. **清理缓存**
   - 菜单栏：File → Invalidate Caches
   - 勾选所有选项
   - 点击 "Invalidate and Restart"
   - 等待 IDE 重启

3. **清理项目**
   - 菜单栏：Build → Clean Project
   - 等待清理完成

4. **重新构建**
   - 菜单栏：Build → Rebuild Project
   - 等待构建完成（可能需要 1-2 分钟）

5. **重新运行**
   - 点击运行按钮（绿色三角形）或按 Shift+F10
   - 等待预览器启动

### 方法 2: 快速重启预览器

如果方法 1 太慢，可以尝试：

1. **停止当前预览器**
   - 点击停止按钮

2. **删除 .preview 缓存**
   - 关闭 DevEco Studio
   - 删除 `help_system/entry/.preview` 文件夹
   - 重新打开 DevEco Studio

3. **重新运行**
   - 点击运行按钮

### 方法 3: 使用 PowerShell 脚本（最快）

运行已有的重建脚本：

```powershell
cd help_system
.\rebuild.ps1
```

然后在 DevEco Studio 中点击运行。

## 常见问题

### Q: 重新编译后还是报错？
A: 尝试以下步骤：
1. 完全关闭 DevEco Studio
2. 删除以下文件夹：
   - `help_system/.hvigor`
   - `help_system/entry/.preview`
   - `help_system/entry/build`
3. 重新打开 DevEco Studio
4. 执行 Build → Rebuild Project

### Q: 编译时间太长？
A: 这是正常的，首次编译新文件需要：
- 解析 ArkTS 代码
- 生成模块映射
- 构建预览器资源
- 通常需要 1-2 分钟

### Q: 预览器启动失败？
A: 检查：
1. 是否有其他预览器实例在运行
2. 端口是否被占用
3. 尝试重启 DevEco Studio

## 技术说明

HarmonyOS 预览器使用模块缓存机制：
- 新创建的文件不会自动加入模块映射
- 需要重新构建才能生成正确的模块引用
- `.preview` 文件夹存储预览器缓存
- `build` 文件夹存储编译产物
