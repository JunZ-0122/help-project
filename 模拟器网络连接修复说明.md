# 模拟器网络连接修复说明

## 问题描述

HarmonyOS模拟器显示 "Couldn't connect to server" 错误，无法连接到后端服务。

## 问题原因

模拟器无法访问 `localhost:8080`，因为：
- `localhost` 在模拟器中指向模拟器自己，而不是宿主机（你的电脑）
- 需要使用宿主机的实际IP地址

## 解决方案

### 1. 修改API配置

已将以下文件中的 `localhost` 改为电脑IP地址 `172.31.32.1`：

- ✅ `help_system/entry/src/main/ets/services/ApiService.ets`
  ```typescript
  const API_BASE_URL = 'http://172.31.32.1:8080/api';
  ```

- ✅ `help_system/entry/src/main/ets/services/WebSocketService.ets`
  ```typescript
  const WS_BASE_URL = 'ws://172.31.32.1:8080/ws/chat';
  ```

### 2. 配置Windows防火墙（可选）

如果模拟器仍然无法连接，需要配置防火墙：

**方法1：使用脚本（推荐）**
1. 右键点击 `help/enable_firewall.ps1`
2. 选择"以管理员身份运行"
3. 等待脚本执行完成

**方法2：手动配置**
1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 选择"入站规则" → "新建规则"
4. 选择"端口" → "TCP" → "特定本地端口：8080"
5. 选择"允许连接"
6. 应用到所有配置文件
7. 命名为"Spring Boot 8080"

### 3. 重新编译前端应用

在DevEco Studio中：
1. 点击 Build → Clean Project
2. 点击 Build → Rebuild Project
3. 重新运行应用

或使用命令行：
```powershell
cd help_system
.\rebuild.ps1
```

### 4. 验证连接

后端服务已验证可以通过IP访问：
```
✓ 后端地址: http://172.31.32.1:8080
✓ API测试: 成功（登录接口返回200）
✓ 端口状态: 监听中
```

## 注意事项

### 真实设备测试

如果要在真实HarmonyOS设备上测试：
1. 确保设备和电脑在同一WiFi网络
2. 使用相同的IP地址配置（172.31.32.1）
3. 如果IP地址变化，需要重新修改配置

### IP地址变化

如果电脑IP地址发生变化：
1. 运行 `ipconfig` 查看新IP
2. 修改 `ApiService.ets` 和 `WebSocketService.ets` 中的IP地址
3. 重新编译应用

### 网络切换

- 使用WiFi时：IP通常是 `192.168.x.x`
- 使用以太网时：IP可能是 `172.x.x.x` 或 `10.x.x.x`
- 使用虚拟网卡时：可能有多个IP地址

当前使用的IP：`172.31.32.1`（可能是虚拟网卡或以太网）

## 故障排查

如果仍然无法连接：

1. **检查后端服务**
   ```powershell
   curl http://172.31.32.1:8080/api/auth/login
   ```
   应该返回错误（因为没有提供登录信息），但不应该是连接错误

2. **检查防火墙**
   ```powershell
   netsh advfirewall firewall show rule name="Spring Boot 8080"
   ```
   应该显示规则存在

3. **检查端口监听**
   ```powershell
   netstat -ano | findstr :8080
   ```
   应该显示端口正在监听

4. **检查模拟器网络**
   - 在模拟器设置中检查网络连接
   - 尝试在模拟器浏览器中访问 `http://172.31.32.1:8080`
